# 次段: α×β 少数組合せ比較へ進めるか（決裁メモ）

**作成日**: 2026-04-08  
**性質**: **決裁メモ（コード変更なし）**。根拠: `docs/PAYROLL_BUDGET_POSTOFF_RATIO_TUNING_RESULTS_2026-04.md`。計画: `docs/PAYROLL_BUDGET_POSTOFF_RATIO_TUNING_PLAN_2026-04.md`。前段決裁: `docs/PAYROLL_BUDGET_POSTOFF_RATIO_DECISION_2026-04.md`。BUFFER 観測: `docs/PAYROLL_BUDGET_POSTOFF_BUFFER_TUNING_RESULTS_2026-04.md`。

---

## 1. 目的

- **RATIO（α）1.0 / 1.05 / 1.10 観測**を受け、**次段の比較単位**を **「α 単独の微調整継続」**と **「α×β の少数組合せ比較」**のどちらを主とするか **1 本に決める**。
- **本メモは実装・係数確定ではない**。

---

## 2. 確定事実（観測の要約）

- **α=1.0（β=3M 固定）**: **`gap_unique=1`**、**`room_unique=1`**、**`pre_le_room=0`**、**user `gap=3M`** — **分布は潰れたまま**（BUFFER-only 時と同型）。
- **α=1.05**: **`gap_unique=48`**。**`room_unique`** が **D2=46 / D1=48** と **動き始めた**。**`pre_le_room=0` は維持**。
- **α=1.10**: **before の `gap` レンジはさらに拡大**。**`room_unique=48`**。**`pre_le_room`** が **D2=958 / D1=72** と **0 から変化**。
- **全段階・両 save 共通**: **`final_offer > buffer = 1920/1920`** — **before 側の改善と、最終 offer 分布の改善はまだ同義ではない**。
- **読み取り**: **α 比較は有効**だったが、**最終的な係数判断には、まだ一段（別軸の組合せ）が足りる**段階に入った。

---

## 3. 今回の判断（1 案）

**次段は、α 単独比較を細かく刻み続けるのではなく、α×β の少数組合せ比較へ進める。**

- **探索は広げない**。**α は有望帯に絞る**（**`α=1.05` を軸候補**、**`α=1.10` を強め候補**として扱う方向でよい）。**β も少数候補だけ**を掛ける。
- **総当たり・格子の大量化はしない**（**次メモでセルを数え、上限を明記**する）。
- **断定しないこと**: **最終 α / β**、**即コード変更**。

---

## 4. この判断の理由

- **α 単独で before 分布が動くこと**は **確認済み**。**1.05 と 1.10 の差**も **取れた**。
- **次に自然なのは**、**β を動かしたときと**の **組合せ**で **「どこまでで十分か」**を見ること（**1 軸だけの微調整を延長するより効率的**）。
- 一方 **final_offer 飽和は不変**なので、**別論点は残る** — **だからこそ**、**α を 1.06 / 1.07… と増やす前に**、**少数組合せで一度まとめて切る**方が **観測コストに見合う**。

---

## 5. 非目的

- **本メモによるコード変更**。
- **α / β の最終値の断定**。
- **`clip` / `λ` / FA `buffer` の変更**。
- **α を 1.06 / 1.07… のように細かく刻む**ことを **次段の主戦術**とすること。
- **総当たり観測の開始**（**セル数の上限なし**）。
