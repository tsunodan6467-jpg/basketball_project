# FA：`payroll_budget` クリップ線形緩和 λ の第一試行決裁メモ

**作成日**: 2026-04-06  
**文書の性質**: **意思決定メモ（コード変更なし）**。式タイプ: `docs/FA_PAYROLL_BUDGET_CLIP_FORMULA_OPTIONS_NOTE_2026-04.md`。クリップ構造: `docs/FA_ROOM_TO_BUDGET_CLIP_REDESIGN_NOTE_2026-04.md`。実装: `basketball_sim/systems/free_agency.py` の `_PAYROLL_BUDGET_CLIP_LAMBDA`・`_clip_offer_to_payroll_budget`。観測: `tools/fa_offer_real_distribution_observer.py` 等。

---

## 1. 文書の目的

**λ = 0**（現行 `min(offer, room)` 相当）から、**正の λ へ上げる第一試行**で **どの値から始めるか**を1つに決め、次コミットの入力とする。**最終的な理想 λ**の確定は目的としない。

---

## 2. λ 第一試行の意味

- **現状**: **λ = 0** のとき **`offer > room > 0`** で `clipped = room`、全体として **従来の `min(offer, room)` と同値**（テストで固定済み）。  
- **λ > 0**: 同条件下で **`clipped = room + round(λ * (offer - room))`** となり、**room 天井から芯方向へ一部だけ戻す**（全面張り付きの緩和）。  
- **目的**: **`final_offer == buffer` 型の全面張り付きを、交渉芯側へわずかに開く**こと。**`payroll_budget` / 経営ストーリーの完全解除ではない**（ソフト化の第一歩）。  
- **小さく始める理由**: 交渉額が **名義 room を超えうる**解釈に触れるため、**経営・CPU・手動 FA への波及を段階的に見る**必要がある。

---

## 3. 候補値の比較（0.1 / 0.2 / 0.3）

| λ | 実装安全性 | 説明のしやすさ | 張り付き緩和の見込み | 経営制約を崩しすぎないか | テスト・観測しやすさ |
|---|------------|----------------|----------------------|--------------------------|----------------------|
| **0.1** | 高 | 高（「ギャップの1割を戻す」） | **控えめ**（行列によっては差が小さく見える可能性） | **最も穏当** | 高（差分が小さく、回帰の切り分けが容易） |
| **0.2** | 高 | 高（「2割」） | **中**（観測で変化が出やすい） | 中（0.1 より芯寄り） | 高 |
| **0.3** | 高 | 高 | **やや強い** | **初手としてはやや攻め**（同じ式でも係数が大きい） | 高 |

**補足**: `offer - room` が大きいペアでは、**0.1 でも絶対額の移動は無視できない**一方、**ギャップが小さいペアでは 0.1 の効果が観測上ノイズに近づきうる**。初手は **安全側を優先**し、**効きが薄い場合は第二試行で 0.2** を検討するのがよい。

---

## 4. 推奨する第一試行値

**第一試行として λ = 0.1 を採用する。**

- **理由**: **最小正の一歩**として **経営制約ストーリーからの逸脱が最も小さく**、**ロールバック（λ を 0 に戻す）も説明しやすい**。**「全面張り付き緩和」の有無を、いきなり大きく振らずに検証**できる。  
- **0.2 を第一候補にしない理由**: 初手としては十分ありうるが、**同じ「第一試行」スロットでは 0.1 の方が安全マージンが大きい**。**効き不足が観測されたら 0.2 を第二試行**とする方が段階が明確。  
- **0.3 を第一候補にしない理由**: **初手として係数がやや強く**、経営・プレイ体感への衝撃が **0.1 / 0.2 より先に大きくなりうる**。
