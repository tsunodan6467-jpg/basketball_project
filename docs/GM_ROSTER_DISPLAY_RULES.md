# GM・人事ロスター表示ルール（正本）

ゲーム内の **人事 → ロスター一覧（Treeview）** と **GM → ロスター／スタメン・ベンチ（テキスト）** で共通の前提を定義する。実装の単一ソースは `basketball_sim/systems/gm_dashboard_text.py`（並び・先発/6th/ベンチの判定）。

## 一覧の並び順（全選手リスト）

1. **ポジション**: PG → SG → SF → PF → C
2. **同一ポジション内**: OVR **降順**
3. **同値**: 選手名（文字列）

`sort_roster_for_gm_view()` がこの順で並べる。

## 役割・記号の対応

| 意味 | 人事「役割」列 | GM「ロスター」テキスト |
|------|----------------|------------------------|
| 先発5人 | `先発` | 行頭 `★` |
| シックスマン | `6th` | 行頭 `6` |
| ベンチ序列 | `控え1` …（ベンチ順の番号） | ベンチブロック内の番号と `6` マーク |

先発・6th・ベンチの**誰が該当するか**は、`get_current_starting_five` / `get_current_sixth_man` / `get_current_bench_order`（いずれも `Team` の `get_starting_five` 等と整合）で決まる。

## ベンチ序列に含まれる条件（フォールバック時）

`get_bench_order_players` が無い場合の算出は **負傷・引退を除く** 控え選手を、上記の **一覧並び** と同じ順で並べたもの（先発5人を除く）。

## 変更操作

- **スタメン1枠の差し替え**は **CLI** の GM メニュー（スタメン変更）と **GUI**（主画面 → GM →「スタメン・ベンチ」タブの **枠／候補／反映（確認）**）のどちらでも可。候補の出し方は `get_available_starting_candidates`（`gm_dashboard_text`）で CLI と一致。
- **カスタムスタメン全体の解除**は同タブの **「自動スタメンに戻す（確認）」**。`Team.clear_starting_lineup()` を実行し、自動選出に戻す（`askokcancel`）。6th はそのまま。
