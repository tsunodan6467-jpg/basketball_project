# 自動役割タグ（表示専用）— パラメータ正本

本ファイルは **自動生成する役割タグ** の判定に使う **しきい値・順位・タイブレーク・優先順位** の **単一正本** である。実装は本ドキュメントを参照し、ここに無い数値を勝手に増やさない（変更時は本ファイルを更新する）。

---

## 1. 目的

- 各選手に **表示専用の 1 枚の役割タグ**（日本語ラベル）を自動で付与する。
- **数値シミュ・交代・起用の正本には使わない**（逆流禁止）。
- 手動の `team_tactics["roles"][pid]["main_role"]`（役割詳細 UI では **「参考役割」**）とは **別系統** として扱う（後述）。

---

| 内部キー案（実装時に採用） | 表示ラベル |

## 3. 優先順位（1 人 1 タグ・先頭一致）

各選手について、**下から上の順に条件を評価し、最初に満たした 1 つのタグだけ**を付与する（上ほど優先度が高い）。

| 順位 | タグ |
|:----:|------|
| 1 | ベンチスコアラー |
| 2 | エース |
| 3 | 司令塔 |
| 4 | シューター |
| 5 | 守護神 / エースストッパー（ポジションで排他） |
| 6 | ロールプレイヤー（最終フォールバック） |

**評価の向き（実装メモ）**: 実装では「優先度 1 の条件を満たすか → 満たさなければ 2 → …」の **分岐チェーン**とするのが読みやすい。

---

## 4. 入力として使う項目（repo 実態）

| 種別 | 項目 | 参照元（repo） |
|------|------|----------------|
| 起用 | 先発 5 人 | `Team` のスタメン（`get_starting_five` 等と整合する既存経路） |
| 起用 | 6th | **`sixth_man_id`（Team 正本）を最優先**（`get_current_sixth_man` 系） |
| ローテ設定 | `target_minutes` | `team_tactics["rotation"]["target_minutes"][str(pid)]`（分は **0〜40 にクランプ**される。`team_tactics._normalize_target_minutes`） |
| 役割（戦術） | `offense_involvement` | `high` / `standard` / `low`（`roles[pid]`） |
| 役割（戦術） | `shot_priority` | `aggressive` / `standard` / `passive` |
| 役割（戦術） | `clutch_priority` | `go_to` / `standard` / `limited` |
| 役割（戦術） | `playmaking_role` | `primary` / `secondary` / `minimal` |
| 役割（戦術） | `defense_assignment` | **`stopper` / `standard` / `light`**（守備タグでは **`stopper` のみ**使用） |
| 能力 | `ovr`, `three`, `defense`, `passing`, `shoot`, `drive`, `rebound` | `Player` の整数属性 |
| ポジション | `position` | `PG` / `SG` / `SF` / `PF` / `C` |

**未確認・明示フィールドなし**: 「ボール保持」専用キーは無い。司令塔判定は上表の `playmaking_role` / `passing` / `offense_involvement` で代替する。

**順位の定義**: 「ロスター全体」「非先発のみ」など表で指定した集合内で、該当ステータス **大きいほど 1 位**（降順ランキング）。同値は **タイブレーク**へ。

---

## 5. タグごとのしきい値表（凍結）

以下、「**順位 k**」= その集合内で 1 位が最良（数値最大が上位）。**先発** = 当該 `Team` の先発 5 人。**非先発** = それ以外。

### 5.1 ベンチスコアラー（優先度 1）

| # | 条件 |
|---|------|
| A | **非先発**である |
| B | （`player_id == sixth_man_id`）**または**（全ロスターで `target_minutes` **順位 ≤ 8** かつ **非先発内**で `target_minutes` **順位 ≤ 3**） |
| C | **非先発内**で **OVR 順位 ≤ 3** |
| D | **非先発内**で **`(shoot + three + drive)` 合計の順位 ≤ 2** |
| E | （`offense_involvement == high`）**または**（`shot_priority == aggressive`）**または**（D の合成が **非先発内で 1 位**） |

成立: **A かつ B かつ C かつ D かつ E**。

### 5.2 エース（優先度 2）

| # | 条件 |
|---|------|
| A | **先発**である |
| B | ロスター全体で **OVR 順位 ≤ 2** |
| C | （全ロスターで `target_minutes` **順位 ≤ 3**）**または**（`offense_involvement == high`）**または**（`clutch_priority == go_to`） |

成立: **A かつ B かつ C**。

### 5.3 司令塔（優先度 3）

| # | 条件 |
|---|------|
| A | （`playmaking_role == primary`）**または**（ロスター全体で **`passing` 順位 ≤ 3** かつ `offense_involvement != low`） |

成立: **A**（優先度 1・2 を通過した上で評価）。

### 5.4 シューター（優先度 4）

| # | 条件 |
|---|------|
| A | ロスター全体で **`three` 順位 ≤ 3** |
| B | （`shot_priority == aggressive`）**または**（**`three >= 62`**） |

成立: **A かつ B**（優先度 1〜3 を通過した上で評価）。

**注**: 論理上「主役タグが付かなかった残りから 3P を見る」が正だが、本正本では実装単純化のため **`three` 順位はロスター全体で算出**する。将来「残り集合のみで再ランク」に変える場合は **本ファイルの改訂差分**として記録すること。

### 5.5 守護神 / エースストッパー（優先度 5）

**守備ブロック（共通）**

| # | 条件 |
|---|------|
| D1 | `defense_assignment == stopper`（**必須**） |
| D2 | ロスター全体で **`defense` 順位 ≤ 4** |

**タグ振り分け**

| タグ | 追加条件 |
|------|----------|
| **守護神** | ポジション **∈ {PF, C}** かつ D1・D2 |
| **エースストッパー** | ポジション **∈ {PG, SG, SF}** かつ D1・D2 |

`standard` / `light` 単独では **守備タグにしない**。

### 5.6 ロールプレイヤー（優先度 6）

優先度 1〜5 の **いずれも不成立** のときに付与する（全員必ず 1 タグ持つ）。

---

## 6. タイブレーク順（同値・同順位時）

**全般（共通の最終）**  
各タグ固有のタイブレークの後、なお同値なら **`player_id` 昇順**（安定・一意）。

### 6.1 ベンチスコアラー

1. `target_minutes` 大  
2. `ovr` 大  
3. `three` 大  
4. `player_id` 昇順  

### 6.2 エース

1. `target_minutes` 大  
2. `clutch_priority` 序数: `go_to` > `standard` > `limited`（実装では 3,2,1 等にマップ可）  
3. `offense_involvement` 序数: `high` > `standard` > `low`  
4. `player_id` 昇順  

### 6.3 司令塔

1. `playmaking_role` 序数: `primary` > `secondary` > `minimal`  
2. `passing` 大  
3. `offense_involvement` 序数（6.2 と同じ）  
4. `player_id` 昇順  

### 6.4 シューター

1. `three` 大  
2. `shot_priority` 序数: `aggressive` > `standard` > `passive`  
3. `shoot` 大  
4. `player_id` 昇順  

### 6.5 守護神 / エースストッパー

1. `defense` 大  
2. **守護神（PF/C）のみ**: `rebound` 大（エースストッパーでは **`rebound` は使わない**）  
3. `target_minutes` 大  
4. `player_id` 昇順  

---

## 7. 守備タグの特記事項

| 項目 | 正本 |
|------|------|
| `defense_assignment` | **`stopper` のみ**が守備タグの前提。`standard` / `light` は守備タグに使わない。 |
| 能力 | **`defense` 順位 ≤ 4**（ロスター全体）。 |
| ポジション | **PF/C → 守護神**、**PG/SG/SF → エースストッパー**（排他）。 |
| `rebound` | **守護神のタイブレーク補助のみ**。エースストッパーでは使わない。 |

---

## 8. ロールプレイヤーの扱い

- 優先度 1〜5 をすべて満たさない選手に **必ず** 付与する。
- **「表示なし」にはしない**（各人ちょうど 1 タグ）。

---

## 9. 使わないもの・境界

| 項目 | 方針 |
|------|------|
| 数値シミュ / Match / `RotationSystem` の内部状態 | **入力に使わない**（表示専用のため）。 |
| rank / 先発 / 6th / `usage_policy` 等 | **起用の正本にはしない**。先発・6th は **タグ判定の入力**としてのみ使う。 |
| 手動 `main_role` | **本自動役割タグとは別物**（役割詳細 UI では **「参考役割」**）。人事ロスター / GM 行末 **「タグ:」** の表示には使わない。既存の保存は互換のため温存。 |

---

## 10. 実装上の注意（末尾）

1. **表示専用**である。本ドキュメントのタグを **契約・キャップ・交代確率・OVR 上書き**等に結びつけない。  
2. **`rank` / 6th / 先発 / `usage` の正本にしない**（読み取り・ラベル付けのみ）。  
3. **`main_role` 手動保存**（`MAIN_ROLE_COMBO_OPTIONS`、役割詳細 UI では **「参考役割」**）とは **別系統**。表示の主役は自動タグ。  
4. **実装時は本ファイルを単一正本**とし、しきい値変更は **PR で本ファイル更新**を必須とする。  
5. `target_minutes` は **0〜40 分クランプ**済みデータを前提とする（正規化は `ensure_team_tactics_on_team` 経路で既存どおり）。
