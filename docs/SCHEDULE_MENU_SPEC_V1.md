# 日程メニュー仕様 第1稿（たたき台）

**関連**: `basketball_sim/models/season.py`（`SeasonEvent`・`round_config`・大会ログ）、`systems/main_menu_view.py`（情報ウィンドウ・メインメニュー）、`main.py`（「日程」＝1ラウンド進行コールバック）。

**位置づけ**: 確定仕様ではない。既存シーズン進行・試合生成を壊さない **読み取り専用の表示メニュー** を前提とする。

---

## 0. 全体所見・保全上の前提

### 0.1 方針

- **大枠**: 「今後の日程」「過去結果」「次戦詳細」を軸にし、**大会種別・ホーム/アウェイは独立メニューにせず、一覧・詳細の共通属性**とする。
- **第1稿の正本は表示のみ**: 日程の手動編集・試合順の変更・大会整合を崩す操作は **スコープ外（禁止）**。
- **データ欠損は正常系**: `None`・空ログ・未知の `competition_type` は「—」「未分類」「取得不可」等で **落とさない**。

### 0.2 現状コード構造とのギャップ（仕様で明示する）

1. **`get_events_for_round(n)` は主にリーグ戦の `SeasonEvent` のみ**  
   全日本カップ・東アジアトップリーグ（内部キー `easl`）・オールアジアトーナメント（内部キー `asia_cl`）等は `simulate_next_round` 内で **別経路** で実行される。同一 API だけでは「そのラウンドの全試合」が揃わない場合がある。

2. **`game_results` のスキーマが限定的**  
   主に `home_team` / `away_team` / スコア系。`competition_type`・ラウンド番号・延長有無・自チーム視点の H/A が **常にあるわけではない**。過去結果のリッチ化は **データ拡張または別ソース** が必要。

3. **「開催日」「次戦までの日数」**  
   `ROUND_CONFIG` に **月**はあるが **実カレンダー日付は未モデル**。第1稿では **「ラウンド N」「◯月頃」「あと K ラウンド」** に寄せる。

4. **メイン UI の「日程」ボタン**  
   現状は **1 ラウンド進行** のコールバック。専用日程ウィンドウを増やしても **進行ロジックと混ぜない**。

### 0.3 危険な前提（避ける）

- 「全大会が `SeasonEvent` 一覧に揃っている」と見なすこと。
- 「`game_results` だけで大会種別・節まで完全再現できる」と見なすこと。
- Season の生成・シミュ順序を、表示のために **大きく書き換える**こと（第1稿では非推奨）。

### 0.4 日程「すべて」の表示補完（実装メモ・2026-03）

`upcoming_rows_for_user_team(..., league_only=False)` は、**SeasonEvent に載らないがラウンドカレンダー上は存在する**内容を、見落とし防止のため **表示のみ** 追記する。

| 補完対象 | 条件（概要） | `event_id` 接頭辞 | 詳細パネル |
|----------|----------------|-------------------|------------|
| 全日本カップ | 杯週・自チームがブラケット上で生存・当ラウンドに `emperor_cup` 行が無い | `emperor_cup_week_r` | 表示補完の注記あり |
| 東アジアトップリーグ（内部キー `easl`） | `easl` ステージあり・自チームが関与見込み・当ラウンドに `easl` 行が無い | `easl_week_r` | 同上 |
| 日本代表ウィンドウ | `national_team_window` 設定あり・当ラウンドに自チーム行がまだ無い | `national_window_r` | 同上 |

**表示補完で済む範囲（当面）**: 上記の「行が無いときに1行足す」ロジックの拡張・文言調整。

**将来、シミュ正本側で検討する範囲**: ディビジョン PO（`playoff`）等を `get_events_for_round` に載せる／進行状態から対戦カードを確定して表示する、アジアカップ本戦・世界一決定戦など **オフ／別フェーズ** の日程を同一ウィンドウにどこまで載せるか。

## 1. 大会表示名と内部識別子

### 1.1 用語の整理

| 種類 | 例 | プレイヤーに見せるか |
|------|-----|---------------------|
| **内部 ID** | `competition_id`（`easl`）、`competition_type`（`easl`）、属性名（`easl_enabled`） | 原則 **見せない**（コード・ログ・デバッグ用） |
| **表示名（正本・本作）** | 下表「ゲーム内表示名（固定）」 | **画面・仕様の正** |

仕様書や実装議論で `easl` と書く場合は **内部キー** を指す。UI 文言は **表示名マッピング** 経由で統一する。

### 1.2 ゲーム内表示名（固定・本作）

| 概念 | ゲーム内表示名（固定） |
|------|------------------------|
| リーグ戦 | **日本リーグ** |
| 天皇杯相当 | **全日本カップ** |
| クラブ東アジア大会（内部キー `easl`） | **東アジアトップリーグ** |
| クラブ・アジア域トーナメント（内部キー `asia_cl`） | **オールアジアトーナメント** |
| 日本代表のアジアカップ（代表ウィンドウ等・`asia_qualifier` / `asia_cup`） | **アジアカップ**（予選／本戦のラベルは `_get_national_team_window_label` 側） |
| インターコンチネンタル相当 | **世界一決定戦** |

### 1.3 内部キー → 表示名（実装時の推奨マッピング）

| competition_id（参考） | competition_type（参考） | 表示名（上表に合わせる） |
|-------------------------|--------------------------|---------------------------|
| `regular_season` | `regular_season` | 日本リーグ |
| `emperor_cup` | `emperor_cup` | 全日本カップ |
| `easl` | `easl` | 東アジアトップリーグ |
| `asia_cl` | `asia_cl` | オールアジアトーナメント |
| `intercontinental_cup` | `intercontinental` | 世界一決定戦 |
| `division_playoffs` | `playoff` | （第1稿）ディビジョン PO 等、別途短縮ラベル可 |
| その他・不明 | — | `—` または `未分類` |

### 1.4 実装メモ

画面向けの大会名は **`competition_display.py` の `competition_display_name`** を正本とする。`Season._build_competitions()` の `Competition.name` は上表と揃える（開発者向けコメント・旧セーブ互換用のキーワードに `EASL` / `ACL` が残る場合がある）。

## 2. 大分類（メニュー構成案）

| ID | 大分類 | 役割 |
|----|--------|------|
| A | **今後の日程（自チーム）** | 残りの自チーム関係試合の一覧。**中心画面**。 |
| B | **過去結果（自チーム）** | 消化済み試合の一覧 → 詳細。 |
| C | **次戦詳細** | 次の1試合（または次ラウンド）の集約。**将来の観戦・戦術ハブ候補**（第1稿は表示中心）。 |
| D | **（非独立）大会カテゴリ・H/A** | A/B/C の **列・バッジ・フィルタ** として持つ。 |

**推奨タブ順（読み流れ）**: C → A → B。実装は **A のみ先行**でもよい。

---

## 3. 各分類の中項目・入力形式

### 3.1 A. 今後の日程

| 中項目 | 入力形式 | 備考 |
|--------|----------|------|
| ラウンド番号 | 表示 | `current_round` 基準で「第何ラウンドの試合か」 |
| 表示用「時期」 | 表示 | `round_config[round].month` 等から **「◯月頃」**（絶対日なし） |
| 対戦相手 | 表示 | 自チーム固定視点 |
| 大会表示名 | 表示 | **§1.2 の固定名**（内部キーはマッピング） |
| 大会カテゴリ（リーグ/カップ/国際） | 表示・フィルタ用 | 表示用タグ。初期は **すべて / リーグのみ** まで |
| H/A | 表示 | 一覧 `H`/`A`、詳細で「ホーム」「アウェイ」 |
| 節／ステージ | 表示 | リーグは「ラウンド N」。カップはログから取れれば表示、無ければ「—」 |
| 次戦まで | 表示 | **「あと K ラウンド」**（日数は第1稿では非推奨または後段） |
| 簡易注目 | 表示（任意） | 連戦・順位差等。**初期は未実装可** |

**操作**: 表示専用。行選択で **同一ウィンドウ内の詳細パネル**。

### 3.2 B. 過去結果

| 中項目 | 入力形式 | 備考 |
|--------|----------|------|
| ラウンド／時期 | 表示 | `game_results` にラウンドが無い場合は **非表示または「順序不明」** を許容 |
| 対戦相手 | 表示 | 自チームでフィルタ |
| 大会表示名 | 表示 | リーグのみなら **日本リーグ** 固定でも可。カップは **後段または別ソース** |
| H/A | 表示 | データが無ければ「不明」 |
| 勝敗・スコア | 表示 | 自チーム視点で W/L |
| 延長 | 表示 | 結果にフラグが無ければ **「—」** |
| 主要スタッツ | 表示 | **将来**（試合サマリー永続化後） |

### 3.3 C. 次戦詳細

| 中項目 | 入力形式 | 備考 |
|--------|----------|------|
| 次の対戦 | 表示 | **次ラウンドの自チーム `SeasonEvent` を主**とし、同一ラウンドのカップは **注記** が現実的 |
| 相手・大会・H/A | 表示 | A と共通の整形関数を再利用 |
| 自チーム／相手順位 | 表示 | `get_standings`。取れなければ「—」 |
| 直近成績 | 表示 | 第1稿は **シーズン累計（勝敗）** の簡易表示でも可 |
| 文脈（連戦・首位争い等） | 表示 | 第1稿 **省略可** |

**将来**: 観戦・戦術へのボタンは **既存メニューへのリンクのみ**（ロジック共有はしない）。

### 3.4 D. 共通表示属性（独立メニューにしない）

- **大会カテゴリ**: 内部で `domestic_league` / `domestic_cup` / `international_club` / `unknown` 等に正規化し、フィルタ・バッジに使う。
- **H/A**: 一覧は短縮、詳細は日本語。
- **フィルタ**: 第1稿は **すべて / リーグのみ**。カップ・国際のみは **一覧データが揃ってから**（空振り防止）。

## 4. 初期実装対象の切り分け

| 区分 | 内容 |
|------|------|
| **初期実装（推奨）** | 自チーム **今後の日程**（`SeasonEvent` + `round_config`）。**表示名は §1.2**。H/A・ラウンド・月ラベル。簡易詳細。 |
| **表示のみ先** | **次戦詳細**（C）は A の先頭行の強調表示までなら低リスク。 |
| **詳細まで（限定的）** | **過去結果**は **リーグ戦分のみ** `game_results` から。欠損は UI で明示。 |
| **後回し** | カップ/EASL の過去を game_results 並みに一覧化、絶対日付、次戦まで○日、リッチスタッツ、高度フィルタ、複雑な「注目試合」ルール。 |
| **将来（Season 非破壊）** | 表示用行の append、または `SeasonEvent` のマージで「そのラウンドの全競技」を可視化（**シミュ順序は変えない**）。 |

## 5. 初期反映イメージ（表示系）

- メインから **「日程」専用ウィンドウ**（経営メニューと同型の読み取り専用）。
- 既存 **情報ウィンドウ**の「次ラウンド予定」は維持。重複回避のため **「詳細は日程ウィンドウ」** 案内1行は任意。
- CLI は変更なしでも可（後から同一純関数を流用可）。

---

## 6. 実装優先順位（保全優先）

1. **`schedule_display` 的な純関数モジュール**（`season` を壊さず、`build_user_team_upcoming_rows(season, team)` 等）。欠損処理・**§1 の表示名マッピング**をここに集約。
2. **GUI は出力の描画のみ**（`main_menu_view` は薄く）。
3. **次戦サマリ**（C の最小）。
4. **過去結果（リーグのみ・欠損明示）**。
5. カップ等は **ログテキストパネル**から段階的に、または **データ拡張後**に一覧へ。

---

## 7. 責務分離（推奨）

| 層 | 責務 |
|----|------|
| **表示用マッピング** | `competition_type` → 固定表示名（§1）。GUI に直書きしない。 |
| **schedule_display（純関数）** | `Season` / `Team` の読み取りのみ。ViewModel 的な行 dict を返す。 |
| **main_menu_view** | ウィンドウ・ツリー・文字列バインド。ビジネスロジックを持たない。 |
| **Season / simulate** | 第1段では **変更しない**（表示で吸収）。 |

## 8. Cursor 視点：懸念・改善・注意

### 8.1 このまま進めてよい点

- 一覧＋詳細、表示専用、編集なし。
- 自チーム視点、H/A・大会表示名の列化。
- `get_events_for_round` と `round_config` を **第1データソース**とする。

### 8.2 曖昧・衝突しやすい点

- 「開催日」を実日付と読める表現にすること。
- `game_results` だけで大会・ラウンドを保証すること。
- 内部キー `easl` をそのままプレイヤー向けに出すこと（**§1 のマッピング必須**）。

### 8.3 先に決めるとよい論点

- 過去結果を「リーグのみ・欠損多め」で出すことの **説明文での許容**。
- **次戦**を「次ラウンドのリーグ戦のみ」と定義するか、同ラウンドのカップは **注記のみ**とするか。

### 8.4 破壊的変更を避ける注意

- `Season.__init__` のイベント生成や `simulate_next_round` の順序を、第1稿の UI のために **変えない**。
- `game_results` 拡張は **append 時のみ**・読みは `dict.get`。
- 表示名変更は **マッピング集中**により、シミュレーション数値に触れない。
