# FA：`payroll_budget` クリップ線形緩和 λ の第二試行決裁メモ

**作成日**: 2026-04-06  
**文書の性質**: **意思決定メモ（コード変更なし）**。第一試行決裁: `docs/FA_PAYROLL_BUDGET_CLIP_LAMBDA_FIRST_TRIAL_DECISION_2026-04.md`。第一試行観測: `docs/FA_PAYROLL_BUDGET_CLIP_LAMBDA_01_PLAYCHECK_2026-04.md`。式: `docs/FA_PAYROLL_BUDGET_CLIP_FORMULA_OPTIONS_NOTE_2026-04.md`。実装: `basketball_sim/systems/free_agency.py` の `_PAYROLL_BUDGET_CLIP_LAMBDA`・`_clip_offer_to_payroll_budget`。観測: `tools/fa_offer_real_distribution_observer.py`、`tools/fa_offer_diagnostic_observer.py`。

---

## 1. 文書の目的

**λ = 0.1** 第一試行の観測（`final_offer > buffer` が **1920/1920** 等）を踏まえ、**第二試行として λ を 0.05 に下げるか**、**0.1 を維持して追加観測を先にするか**を1本に決め、次コミットの入力とする。

---

## 2. λ=0.1 の評価

- **張り付き破壊には成功した**: `final_offer == buffer` および `0 < final_offer < buffer` が **0**、`final == buffer` 型の **全面張り付きは解消**（`docs/FA_PAYROLL_BUDGET_CLIP_LAMBDA_01_PLAYCHECK_2026-04.md` と整合）。  
- **観測系では `final_offer > buffer` が全面化**: **1920/1920**、quick / quicksave / `--seasons 1` で **同型**。**別プレートへの全面同一パターン**であり、行列上は **帯が豊かに開いたとは言い切れない**。  
- **よって「効いた」が「強め」の可能性はかなり明確**。第一試行として **意図（room から芯へわずかに戻す）は達成**したが、**この FA×チーム行列では緩和が一方向に寄り切っている**。

---

## 3. 0.05 へ下げる案

**内容**: `clipped = room + round(λ * (offer - room))` において **λ を半減**し、**room を超える幅を緩やかにする**。

| 長所 | 短所 |
|------|------|
| **`> buffer` 全面化が弱まり**、`0 < final ≤ buffer` 帯や **buffer 近傍**が観測上 **再び現れる可能性**がある | **ギャップが小さいペアでは差が観測しにくい**、行列が依然 **offer ≫ room 支配**なら **1920/1920 のまま**の可能性 |
| **第一試行（0.1）との対比が明確**（半分の係数で効きの感度を見る） | **「弱すぎ」と判断されれば第三試行の議論**に進む |
| **実装は定数1行中心**で済み、回帰・観測フローは既存のまま | 経営・説明文の論点は **λ に依存**するため、値の探索は **段階的に続く** |

---

## 4. 0.1 維持＋追加観測案

**内容**: **`_PAYROLL_BUDGET_CLIP_LAMBDA = 0.1` を固定**し、**実セーブ種別・シーズン進行・手動 FA 体感**など **観測条件を増やす**。

| 長所 | 短所 |
|------|------|
| **0.1 が「本当にプレイ上強すぎるか」**を、行列以外の文脈で **補強判断**できる | **意思決定が1段先送り**になり、**1920/1920 が式の強さシグナルである限り**、λ 感度の **次データ点が得られない** |
| **定数を増やさず**にリスク認識だけ深められる | 追加観測だけでは **`> buffer` 一色の行列**は **自発的には変わらない**（観測対象の違いによる見え方の差に留まる） |

---

## 5. 推奨判断

**第一候補: 第二試行として λ を 0.05 に下げる（実装は次コミットで定数1行＋既存テスト／観測の更新）。**

**理由（要約）**:

1. **観測行列で `final_offer > buffer` が 1920/1920** は、**0.1 がこの装置上「強く効いている」**ことの強いシグナル。**次に欲しい情報は「半分の λ で帯が戻るか／全面化が続くか」**であり、**追加の実セーブ観測だけではその分離が弱い**。  
2. **コストが低い**（定数1行＋既存パイプライン）。**0.05 でまだ全面化なら**、問題は **λ より行列の構造（offer と room の関係）**側に寄っていると切り分けやすい。  
3. **0.1 維持＋観測**は、**並行で軽く足してよい**が、**λ 感度の決裁を先送りする主路線にはしない**のがよい（第一試行の目的は **張り付き緩和の段階探索**であり、**すでに 0.1 の効きは観測済み**）。

**慎重論の位置づけ**: 経営ストーリーや説明文を **いま絶対に動かしたくない**場合のみ、**短い追加観測を挟んでから 0.05** でもよいが、**本メモの第一候補は 0.05 第二試行**とする。
