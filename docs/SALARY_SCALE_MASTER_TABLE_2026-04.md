# 年俸スケール正本テーブル案

**作成日**: 2026-04-06  
**文書の性質**: **設計正本（案）**。数値の最終確定・定数の確定・実装コミットは本書の範囲外。実装時は別タスクで本書を参照し、差分は変更履歴に残す。  
**前提となる監査**: `docs/SALARY_SCALE_AUDIT_2026-04.md`

## 1. 文書の目的

本書は、国内バスケ GM シミュレーションにおける **年俸の「世界観」を一本化するための正本テーブル案**である。ディビジョン別のクラブ総年俸帯、選手層別の個人年俸帯、属性に基づく補正の考え方、および初期ロスター・FA・再契約・CPU 本格 FA といった **各システムを将来どうこの正本へ接続するか**の方向性を定義する。

目的は **実装をこの文書だけで完結させることではない**。今後の係数調整・リファクタ・バランス作業の **合意形成と優先順位付け**に使うことである。

---

## 2. この文書が解決したい問題

監査で確認された事実として、次が並立している。

- 開幕ロスターは **`GENERATOR_INITIAL_SALARY_BASE_PER_OVR`（OVR×122万円オーダー）** で生成され、**D1〜D3 で総年俸がほぼ同じ帯（約10〜11億/チーム）**に集中しうる。
- FA 目安 **`estimate_fa_market_value`** は **`ovr * 12000` 核**であり、ロスター年俸と **桁が約100倍ずれる**。
- オフ手動 FA の局所調整だけでは、**リーグ全体の物差し**は揃わない。

つまり問題は「どの画面の FA が安い」ではなく、**年俸スケールの正本が未定義のまま、複数の独立した式が共存している**ことである。本書は、その **正本をテーブルとして先に置く**ことで、以降の実装が **同じレンジ表を参照する**状態を目指す。

---

## 3. 年俸スケール設計の基本思想

年俸は **どの画面でも同じ世界観の数値**であるべきである。初期ロスター、FA、再契約、CPU 本格 FA が **別々の物差し**を持ち続ける状態は、プレイヤーの比較判断と説明責任の両方を損なうため、**段階的にやめる**。

設計の順序は次を採用する。**まずクラブ総年俸帯（ディビジョン×格）を決め、その下に個人年俸帯をぶら下げる。** 個人だけを先に細かく決めても、クラブ全体のペイロールと整合しないため、**トップダウンでレンジを固定し、ロスター構成や希少性で個人に配分する**イメージである。

現実のプロリーグを参考にするが、**最優先はゲームとしての成長感・格差感・長期運営の面白さ**である。現実再現は手段であり、数値の完全一致は求めない。

**D3 ボトムで総年俸 4億前後**は、「弱小スタートとして苦しいが、昇格や補強で希望が見える」水準として **有力な案**である。プレイヤーに **明確な下枠**を与えつつ、目標レンジへの成長余地を残す。

**サラリーキャップ 12億**は、初期から全クラブを cap で縛りすぎず **窮屈さを避けつつ**、長期プレイでのインフレ・クラブ成長の結果として **上位層・D1 で cap が効いてくる余地**を残す設計として **有力な案**である。

## 4. ディビジョン別クラブ総年俸帯テーブル

以下は **ゲームバランス優先で採用する案**であり、実装時の定数そのものの確定ではない。レンジは **年間ペイロール合計（円）**の想定である。

| 区分 | 想定総年俸レンジ（円/年） | 想定クラブ像 | ゲーム上の意味 |
|------|---------------------------|--------------|----------------|
| **D1 トップ** | 約 **11億〜12億**（cap 近傍） | 優勝争常連・人気・資金力の象徴クラブ | プレイオフ上位・タイトル期待。**cap を意識する最初の層**として説明しやすい。 |
| **D1 ミドル** | 約 **9.5億〜10.5億** | プレイオフ圏を狙う堅実な強豪 | 「勝てるが王手はかけにくい」。**安定志向プレイ**の拠点になりうる。 |
| **D1 ボトム / D2 トップ** | 約 **7.5億〜8.5億** | D1 で苦戦するか、D2 頭一つ抜けたクラブ | **降格・昇格ドラマ**の交差点。補強一発で上を狙う張り合い。 |
| **D2 ミドル** | 約 **6.5億〜7.5億** | 中位安定〜プレイオフの縁 | D1 昇格を目指す**中核層**。差は小さくてよいが帯は識別可能に。 |
| **D2 ボトム / D3 トップ** | 約 **5.5億〜6.5億** | D2 下位〜D3 上位の跳ね上がり候補 | **昇格・降格の物語**の受け皿。 |
| **D3 ミドル** | 約 **4.5億〜5.5億** | 中堅どまりの二部相当 | プレイオフ争いはするが資金は限られる。 |
| **D3 ボトム（開始時ユーザーチーム想定）** | 約 **3.8億〜4.2億**（目安 **4億**） | 資金・人材ともに薄いが、若手と方針で這い上がれる | **新規プレイの「底」**として明確にし、成長カーブの起点にする。 |

**なぜこのレンジがゲームとして自然か**（要約）: cap 12億を **上限の「天井」**として置きつつ、D1 トップだけがそこに手が届く／届きかけるイメージにすると、**長期で「強くなるほど cap に触れる」**緊張感を説明できる。D3 ボトムを 4億付近に置くと、**ディビジョン間で約2〜3倍の総年俸差**が出せ、**昇格・降格のインパクト**が数値でも伝わりやすい。現実のプロリーグの厳密な会計一致は求めず、**格差がプレイの選択に効く**ことを優先する。

## 6. 属性補正の方針

同じ OVR 付近でも、**市場価格に差がつくべき理由**を整理する。倍率をここで断定しない。**「何のために補正するか」**を正本として置く。

- **外国籍**: 即戦力・得点・身体能力の期待が高い場合があり、**同 OVR でもプレミアム**が付きうる。一方、枠（最大3人等）があるため **希少性と規制**でバランスする。
- **帰化**: 日本籍として扱うが、獲得経路・ストーリー上 **エース級の価格**になりうる。純日本人エースとの **比較が説明可能**であること。
- **アジア枠**: 枠が限られるため **同能力でも市場が細い**／**獲得競争が局所的**になりうる。補正は「強さ」だけでなく **枠の戦略性**とセットで考える。
- **若手有望株**: 現時点の OVR は低くても **将来値への期待**で年俸が上振れしうる。ただし暴走すると総ペイロールが破綻するため **上限レンジはクラブ帯とセット**で調整する。
- **ベテラン**: ピークは過ぎても **即戦力・リーダーシップ**として契約しうる。**短約・やや低めの年俸**も選択肢として整理する（実装は別途）。
- **ポジション希少性**（必要なら）: ビッグマンや1on1要員など、**リーグ全体の供給が少ないロール**にプレミアムが付きうる。常に適用せず、**極端な偏りがあるときのノブ**として扱う。
- **日本人エース格**: マーケット・人気・代表選出を兼ねた **国内スターのプレミアム**。同 OVR の非日本人との **比較が納得できる説明**を優先する。

これらは **最終的には「base 年俸（市場ベース）」に乗算または加算で効かせる**想定とし、**base 自体を正本テーブルと整合させる**ことが先である。

## 7. 初期ロスター / FA / 再契約 / CPU 本格 FA への接続方針

理想形は、**共通の「市場ベース年俸（base market salary）」**をまず定義し、**文脈ごとの補正**で各システムに接続することである。関数名やモジュール分割は本書では確定しない。**方向性のみ**示す。

| 文脈 | 正本への接続の考え方 |
|------|----------------------|
| **初期ロスター生成** | チームの **想定総年俸レンジ（§4）**を満たすよう、13 人に **個人帯（§5）**を配分する。ディビジョン・シード・人気等で **§4 のどのセルに載せるか**を先に決め、**不足分は若手・控えで埋める**。 |
| **インシーズン FA 表示・CPU 補強** | 選手ごとに **base market salary** を正本から導き、**estimate 表示は base と同オーダー**に揃える。CPU の獲得可否は既存のキャップ・所持金ガードと接続。 |
| **オフ手動 FA** | **本格 FA と同じオファー模型**（`_calculate_offer` 系）を維持しつつ、**入力となる `player.salary` や前提ペイロール**が正本と整合するよう、**プール選手のベース年俸**を正本側で定義した値に寄せる。 |
| **再契約** | **希望年俸・提示額**が **正本の個人帯**と矛盾しない説明になるよう、`desired_salary` のスケールを base と接続。年齢・残留意向は既存ロジックの上に **レンジ逸脱のペナルティ／納得感**を載せる。 |
| **CPU 本格 FA** | `_calculate_offer` は **salary 起点**であるため、**FA プールの選手の `salary` が正本の base と整合**していれば、ログ上のオファーも説明可能になる。 |

**共通の base** を置くことで、**「目安（estimate）」と「オファー（本格 FA）」が別桁になる**状態を、中期的に **同じ円オーダー**へ収束させる。移行は **段階的**でよいが、**最終的には一本の物差し**を正とする。

## 8. いまの実装とのズレ

`docs/SALARY_SCALE_AUDIT_2026-04.md` の要約に沿う。

- **開幕ロスター**: D1〜D3 で **総年俸の差が小さすぎる**（約10〜11億に集中しうる）。§4 の **ディビジョン別格差**が表現されていない。
- **FA estimate**: **`ovr * 12000` 核**であり、ロスター年俸と **桁が合わない**。プレイヤーが **同じ選手を見ても画面で比較不能**。
- **複数物差し**: 生成は **OVR×122万**、契約希望は **OVR×100万**系、FA 目安は **OVR×1.2万**、本格 FA は **`salary` 起点**。**正本テーブルなし**のまま並立している。

本書のテーブル案は、これらの **収束先の像**である。

## 9. 実装順の提案

次の順を推奨する。前工程の合意なしに後工程だけ変えると、再び **局所最適**になるためである。

1. **年俸正本テーブル確定**（本書のレビュー・数値レンジの承認・変更履歴の管理）
2. **初期ロスター生成の是正**（§4・§5 に沿った **総年俸と個人帯の配分**）
3. **FA estimate / インシーズン FA の是正**（base と **同オーダー**へ。表示と説明の一貫）
4. **再契約・CPU 本格 FA の整合**（`salary`・希望額・オファーが **同一世界観**で説明できること）
5. **経営収支・予算スケールの再整合**（年俸が変わると **所持金・ラウンド収入・オフ補填**のバランスも見直す）

理由の要約: **世界の「土台」であるロスターと base** を先に揃えないと、FA だけ直しても **比較の基準がブレる**。経済は **年俸スケール確定後**にノブを合わせる方が手戻りが少ない。

## 10. 次に最小差分で触るべき対象（1つだけ）

**初期ロスター生成の年俸・ペイロール配分を、ディビジョン別総年俸帯（§4）が再現されるように見直す設計・実装タスク**を、次の一手とする。

**理由（FA estimate より先に初期を触る根拠）**:

- プレイヤーが最も長く見る **ベースラインの年俸分布**は **開幕ロスター**である。ここが §4・§5 と無関係だと、**FA だけ正本に寄せても「チームの星」と「FA表示」が並べて比較できない**期間が残る。
- 監査で示された **100倍級の乖離**の一方の極は **`estimate_fa_market_value`** だが、もう一方の極は **生成による高い `salary`** である。**総ペイロールの帯を先に制御**すると、以降 FA・再契約は **「このクラブ・この層ではいくらが妥当か」**の議論に落とせる。
- **d995b48 等の暫定補正**は、正本が立ったあと **縮小・削除できるか**を判断しやすい。

逆に **FA estimate だけを先に全面改修**すると、開幕ロスターが依然 **平坦な総年俸**のままでは、**ゲーム全体の格差設計**が後追いになり、テストと説明が二重になる。
