# 観測 salary と `ovr` 帯の対応（照合メモ）

**作成日**: 2026-04-08  
**性質**: **主水準＝線形項との突き合わせ（コード変更なし）**。項の読み分け: `docs/FA_MARKET_VALUE_DRIVER_READ_NOTE_2026-04.md`。照合: `docs/FA_MARKET_VALUE_MATCH_NOTE_2026-04.md`。設計焦点: `docs/FA_SALARY_DESIGN_FOCUS_DECISION_2026-04.md`。係数: `GENERATOR_INITIAL_SALARY_BASE_PER_OVR = 1_220_000`（`game_constants`）。

---

## 1. 目的

- **Cell B** で見えている **`player_salary` / `base`（約 85M〜115M 級）**が、**主水準 `ovr × 1_220_000`（＋`legacy_floor` の max）**だけで見たとき **どの `ovr` 帯に相当しやすいか**を **帯の目安**で **固定**する。  
- **厳密な行再計算・原因断定・修正案はしない**。

---

## 2. 確定事実

- **Cell B・`pre_le_pop`**: **`player_salary ≒ base`**。**D2** で 約 **85M〜110M**、**D1** で 約 **89M〜115M**（**観測要約**）。  
- **主水準（A）**: **`ovr * GENERATOR_INITIAL_SALARY_BASE_PER_OVR`** と **`legacy_floor`** の **max**（**`FA_MARKET_VALUE_DRIVER_READ_NOTE` どおり**）。  
- **上振れ（B）**: **`potential`**（**スケール後、S でおよそ 2.5 千万円強の加算オーダー**。**実装の整数除算どおり**）。  
- **補助（C）**: **`age` / `fa_years_waiting`**（**数千万円オーダーの加減**。**帯全体の桁は主水準側**）。

---

## 3. 今回の照合

**線形項のみのざっくり対応**（**`salary ≈ ovr × 1_220_000` とみなした逆算**。**床・`potential`・年齢・待機は未加算**）

| 目安 salary | 対応 `ovr`（線形のみ・近似的） |
|-------------|----------------------------------|
| **85M** | **約 70** |
| **95M** | **約 78** |
| **105M** | **約 86** |
| **115M** | **約 94** |

**観測レンジとの読み**

- **下限側（85M〜89M）**は **線形だけでおよそ `ovr` 70 台前半〜中盤**。**`potential` が C 近辺で、補正がややマイナス**なら **同帯のやや下**もありうる（**厳密には再計算していない**）。  
- **上限側（110M〜115M）**は **線形だけでおよそ `ovr` 90 台前半〜中盤**。  
- **`potential` の上振れ（S/A 等）を数千万円〜約 2.5 千万円足すイメージ**で載せると、**同じ観測上限に届くのに必要な `ovr` は、線形のみの目安よりわずかに下がりうる**（**「半段〜1段」下げ、程度の目安**。**断定しない**）。

---

## 4. 今回の判断（1 案）

**Cell B の 85M〜115M 級 salary は、主に一定以上の `ovr` 帯で自然に発生している読みが強い。したがって次段は、`estimate_fa_market_value` の係数調整そのものより先に、「どの `ovr` 帯を高すぎと見るか」を読む。**

- **`potential` を永久に第2要因と固定**は **しない**（**上振れは依然として無視しにくい**）。  
- **係数いじり**は **「帯の議論」のあと**でも **遅くない**、という **順序の提案**に **留める**。
