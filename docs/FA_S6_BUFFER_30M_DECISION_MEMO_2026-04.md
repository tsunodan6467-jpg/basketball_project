# S6 緩和：buffer 第三試行（3000万円）決裁メモ

**作成日**: 2026-04-06  
**文書の性質**: **意思決定メモ（コード変更なし）**。第二試行決裁・観測: `docs/FA_S6_BUFFER_10M_DECISION_MEMO_2026-04.md`、`docs/FA_S6_BUFFER_10M_PLAYCHECK_2026-04.md`、`docs/FA_S6_BUFFER_10M_REALWORLD_PLAYCHECK_2026-04.md`。第一試行観測: `docs/FA_S6_BUFFER_3M_PLAYCHECK_2026-04.md`。設計枠: `docs/OFFSEASON_FA_PAYROLL_BUDGET_BUFFER_PLAN_2026-04.md`。実装正本: `basketball_sim/models/offseason.py` の `_OFFSEASON_FA_PAYROLL_BUDGET_BUFFER`。観測スクリプト: `tools/fa_offer_real_distribution_observer.py`（stdout に `final_offer == buffer` / `0 < final < buffer` / `final > buffer` を表示）。

## 1. 文書の目的

**1000万円 buffer 第二試行**適用後も、主要観測条件で **S6 の `room` 天井＝`final_offer` の全面張り付き**が **1000万円にスケールしただけ**であることが確認済み。本メモは **第三試行として buffer を 3000万円に上げるか**を **段階試行の可否**として決裁し、次コミットの入力にする。**最終的な「理想 buffer 額」**の確定は目的としない。

---

## 2. 10,000,000 円第二試行の評価

- **達成**: **3M 第一試行の「全面 3M 張り付き」から、天井額は 10M に引き上がった**（決裁どおり第二試行はコード上反映済み）。**3M 未満への張り付きは観測系では解消**（`0 < final <= 3M` は 0）。  
- **観測（`fa_offer_real_distribution_observer.py`）**: **`quicksave.sav`（`--save`）**および **クイック生成＋`--seasons 1`** のいずれでも、総サンプル **1920** に対し  
  - **`final_offer == buffer`（10,000,000）: 1920 / 1920**  
  - **`0 < final_offer < 10,000,000`: 0**  
  - **`final_offer > 10,000,000`: 0**  
  が **両条件で同型**に確認できた（stdout 改修後、厳密件数が直接読める）。  
- **総評**: **第二試行として有効**（桁の引き上げ・説明可能性の改善）。ただし **「buffer 天井＝オファー天井」が行列全体を占める構造は維持**され、**10M は観測上の終着ではない**。**次ノブ（第三試行）を検討する前提は十分に満たしている**。

## 3. 30,000,000 円第三試行案

| 観点 | 内容 |
|------|------|
| **効果** | `room_to_budget` の典型天井が **10M → 30M** に上がり、**中〜高額 FA の芯**が **10M クリップから外れる余地**が大きくなる見込み。億超級は **依然クリップ**されうる。 |
| **実装** | **`offseason.py` の `_OFFSEASON_FA_PAYROLL_BUDGET_BUFFER` を 30,000,000 に変更**する最小差分が中心（`Offseason.run` の同期呼び出し位置は据え置き）。**関連テスト・観測スクリプトの期待値／表示ラベル**の整合が伴う（第二試行と同型）。 |
| **リスク** | **10M より経営ガイドライン・オーナーミッション・CPU FA オファー分布への影響はさらに大きい**。ただし **`max(既存, roster + B)`** の枠・**S1（`soft_cap_early`）経路**は従来どおり分離。 |
| **根拠の十分性** | **実セーブ（quicksave）と 1 シーズン後の合成世界の両方**で **全面 10M 張り付き**が再現しており、**「観測装置が示す次の段階」として B を上げる**合理性は高い。**最終額の確定ではなく段階試行**という位置づけと整合。 |

**追加観測の位置づけ**: **複数セーブ・長期プレイ**では分布が異なる可能性はあるが、**第二試行後の主要2条件がいずれも同型**である以上、**第三試行を無期限保留するほどの反証は現時点ではない**。実装後に **30M 適用の playcheck を1回記録**するのは推奨。

---

## 4. 推奨判断

- **3000万円を第三試行の第一候補として採ってよい**（＝**据え置きより前進が合理的**）。  
- **理由**: (1) **10M も観測上「終着」になっていない**（全面張り付きが **実セーブ路・シーズン1路の両方**で確認済み）。(2) **変更経路が局所**（定数＋テスト・観測整合）でロールバックも容易。(3) **3M → 10M → 30M** は **段階試行として連続**。(4) **S1 決裁**および **`_calculate_offer` 本体**に手を入れずに進められる。  
- **慎重線**: **経営バランスの最終評価**はプレイ全体に依存するため、**第三試行実装後は必ず `fa_offer_real_distribution_observer` を再実行し、`final == buffer` 比率の変化を記録**する。**それでも全面張り付きが続く場合**は **第四段ノブ**（定数または設計見直し）を別途検討する。
