スクロール部は **静的カードなし**。表見出し・選手行は **100% GDScript 生成**。

  - StatusLabel（RootCol 直下・エラー時表示・赤系）
  - Scroll (VBox RowList へ)
    - RowList (%RowList): separation 8 のみ。**子ノードはシーンに無し**（実行時に .gd が追加）

SubResource:

  - StyleBoxFlat_chip … バッジ2つのみ

4. roster_view.gd の動的表示処理
--------------------------------------------------------------------------------
- _apply_snapshot: JSON 成否で StatusLabel / ヘッダー各 Label テキストを更新。
- _clear_rows: %RowList の子をすべて queue_free。
- _add_table_header_row:
  - HBoxContainer.new() + 各列 Label.new()
  - **add_theme_color_override("font_color", Color(0.82, 0.88, 0.96))** … 暗背景用の明色
  - font_size 12、列幅 _col_width
  - 続けて HSeparator を RowList に追加
- _add_player_row:
  - 同様に HBoxContainer + 9 セルの Label.new()
  - **add_theme_color_override("font_color", Color(0.92, 0.94, 0.98))** … データ行も明色
  - 選手名列・状態列に tooltip / mouse_filter の特例あり

コメントより「背景・テーマ追加はしない」と明示されているが、現状は暗 Background 直置きの表なので明色が妥当。


5. Theme 適用しやすい箇所
--------------------------------------------------------------------------------
**A. ヘッダー限定（最も契約人事と同型）**
- ext_resource で phase4_readonly_core.tres を追加し、RosterView に theme を設定。
- HeaderCard: theme_type_variation = Phase4HeaderCard、**StyleBoxFlat_header の panel override を削除**。
- チップは暗背景のまま → ReadonlyBadge / ModeStrip の chip override とラベル明色は **維持で可**。
- TitleLabel / TeamNameLabel / SummaryMetaLabel / DataSourceLabel / FooterNoteLabel を
  契約人事ヘッダーと同系の **濃色 override** に差し替え（白アイボリー上で可読に）。
- HomeNavButton は Theme の Button スタイルが自動適用される。

**B. ルート Theme のみ先に入れる**
- ヘッダー panel はまだローカル override のままにしてもよいが、二重管理になるため非推奨。

**C. スクロール周りを「カード化」する**
- RowList を **PanelContainer + Phase4SummaryCard** で包む案は、**ノード階層が1段増える**（構造変更）。
- 白カード内に表を載せるなら、**現状の明色セルは不可読** → 必ず **.gd で色または variation 変更**が必要。


6. Theme 適用が危険な箇所
--------------------------------------------------------------------------------
- **表ヘッダー・データ行を白カード上に載せる**のに .gd の明色 override を残すこと
  → コントラスト崩壊。
- Header を Phase4HeaderCard にしただけで **TitleLabel 等の明色 override を残す**こと
  → 契約人事で起きていたのと同様に薄く見える。
- StatusLabel は暗背景上のエラー表示のため、**現状の赤系のまま**が無難（ヘッダー外）。


7. ヘッダー限定適用案
--------------------------------------------------------------------------------
- 変更は主に roster_view.tscn + load_steps 増（Theme ext_resource）。
- StyleBoxFlat_header を参照しなくなれば **未参照 SubResource 削除可能**だが、削除は
  Godot 再保存で差分が広がることがあるため、**別コミットで削除**するか、残しても害は小さい。
- 契約人事の 4c1cb08 系の色（タイトル 0.08,0.11,0.18、メタ 0.2,0.25,0.36、DataSource 0.18,0.28,0.55 等）
  を **FooterNoteLabel** にも流用すれば一貫する。


8. カード限定適用案
--------------------------------------------------------------------------------
- 現状シーンに **静的 Summary / Warning カードは無い**。
- 「カード」を足すには PanelContainer を新設する必要があり、**純粋に .tscn のみ**でも
  RowList の親を差し替える **階層変更**になる。
- カード内に表を入れるなら **.gd 必須**（セル色・または theme_type_variation 付与）。


9. 表 / 選手行への適用案
--------------------------------------------------------------------------------
- **現状のまま暗背景上**なら、Theme の **Phase4OnDarkTableHead / Phase4OnDarkTableCell** を
  .gd で variation 付与するか、Color を OnDark 系に寄せるのが筋が良い。
- **白 Summary カード内の表**にする場合:
  - Phase4TableHead / Phase4TableCell（濃色）を variation で付与
  - または _add_* 内の Color を Phase4TableCell に合わせた濃色へ変更
- 列幅・HSeparator・tooltip ロジックはそのまま活かせる。


10. .tscn だけで済む最小案
--------------------------------------------------------------------------------
1. ext_resource: res://themes/phase4_readonly_core.tres（uid 既存 Theme と整合）
2. RosterView に theme = ExtResource(...)
3. HeaderCard: **Phase4HeaderCard**（契約人事と同じ。SummaryCard ではない）
4. HeaderCard から theme_override_styles/panel（StyleBoxFlat_header）削除
5. TitleLabel / TeamNameLabel / SummaryMetaLabel / DataSourceLabel / FooterNoteLabel の
   font_color を濃色へ（契約人事ヘッダーと揃えるのが最短）
6. StyleBoxFlat_chip とチップ内ラベルは維持
7. **RowList / Scroll / .gd は触らない** → 表は暗背景のまま明色セルで現状維持

→ **契約人事の「ヘッダー限定適用」手順のほぼコピー**で済む。


11. .gd 変更が必要な案
--------------------------------------------------------------------------------
- 表を **Phase4SummaryCard** の白パネル内に移す（親 Panel 追加）＋セル可読化。
- 表ヘッダー・データの色を **Theme variation** に統一し override をやめる。
- 暗背景のまま OnDark variation に切り替えて Theme 一本化する（見た目微調整は要確認）。

いずれも _add_table_header_row / _add_player_row の修正が中心。


12. UID / 差分リスク
--------------------------------------------------------------------------------
- roster_view.tscn の uid://cr9rosterproto01 は維持すべき。
- ext_resource 追加で **load_steps** が増える（4→5 など）。意図しないノード順の再ソートに注意。
- SubResource 削除時は **差分が広がりやすい** → ヘッダー適用と SubResource 削除を分けるとレビューしやすい。


13. 契約・人事サマリー手順から流用できること
--------------------------------------------------------------------------------
- phase4_readonly_core.tres をルート theme に載せるパターン。
- HeaderCard の **theme_type_variation + ローカル panel StyleBox 削除**。
- ヘッダー直下 Label の **明色 override →濃色**（チップ上ラベルは暗チップ向けに残す）。
- HomeNavButton は追加の scene スタイル無しで Theme の Button が効く。
- **チップは StyleBoxFlat_chip のまま**（契約人事と同じ考え方）。

流用しにくい点:
- 契約人事は Scroll 内に **静的 Panel カード**があったが、ロスターには **無い**。
- 表は **全動的**のため、「カードだけ .tscn」で済むパターンが無い。


14. Cursor としてのおすすめ順
--------------------------------------------------------------------------------
1. **ヘッダー限定**（.tscn のみ・契約人事と同型）… リスク最小・手順検証に最適
2. 表を暗背景のまま **OnDark系 variation + .gd** で Theme に寄せる（見た目は微差）
3. Scroll 内に **Summary カードを1枚追加**し表を包む（.tscn 階層 + .gd 色）
4. 列ごとのリッチ表現（ zebra 等）は **最後**
