3. club_summary の現状生成箇所
--------------------------------------------------------------------------------
**唯一の生成関数**: `basketball_sim/export/home_dashboard_readonly.py`
  - `_club_summary_lines(season, team, rank_record, money, division) -> List[str]`
  - `build_home_dashboard_readonly_dict()` 内で `club_summary = _club_summary_lines(...)`
  - `export_home_dashboard_json_from_world()` → 上記 dict を JSON 化

**Godot 側**: 生成しない。`home_dashboard.gd` は JSON の `club_summary` 配列を読むだけ。

**Tk 本番 UI（参考・Godot export とは別経路）**:
  - `main_menu_view.py` の `_build_club_summary_info()` は **6 行・勝率・直近5・ミッション等**を含む
    **リッチなクラブサマリー**。Godot export の `_club_summary_lines` とは **別設計**。
  - Godot Phase 4 は **MainMenuView をコピーしていない**（簡易 DTO）。

4. club_summary の現状行一覧（from_python export）
--------------------------------------------------------------------------------
`_club_summary_lines` の固定ロジック（最大 6 行、`return lines[:6]`）:

| # | 条件 | 行の内容 | 意味 |
|---|------|----------|------|
| 1 | 常時 | `順位・戦績: {rank_record}` | MetricsRow **CardRank** と同一文字列の再掲 |
| 2 | 常時 | `ディビジョン: {division} / 資金: {money}` | **CardDivision** + **CardMoney** の再掲 |
| 3 | season あり | `シーズン状態: 終了` or `進行中` | レギュラー終了フラグの要約 |
| 3' | season なし | `シーズン状態: セーブにシーズン未同梱（…）` | 年度メニュー直後等 |
| 4 | cap_line != `給与キャップ：未接続` | `_salary_cap_summary(team)` 全文 | トップレベル `salary_cap` キーと **同一関数・同一文字列** |
| 5 | `len(lines) < 4` | `※ 本ブロックは読み取り専用スナップショットです（…）` | パディング注記 |

**典型行数**（season=None・team あり・キャップ計算成功時のプローブ）: **4 行**（1,2,3',4）。
season あり・キャップ成功時も **4 行**（1,2,3,4）。**5 行目の注記は出ない**（len>=4 のため）。

**含まれないもの**（club_summary には出さない）:
  - `owner_trust` → トップレベルキー + **CardTeamExtras** のみ
  - `recent_form` → トップレベルキー + **CardTeamExtras** のみ
  - `warnings` / `tasks` / `news` → 各専用カード

5. MetricsRow との重複
--------------------------------------------------------------------------------
| キー / UI | club_summary での再掲 | 重複度 |
|-----------|----------------------|--------|
| `division` → CardDivision | 行2「ディビジョン: D2」 | **強**（同一値） |
| `rank_record` → CardRank | 行1「順位・戦績: 4位 / 18勝12敗」 | **強**（同一文字列） |
| `money` → CardMoney | 行2「資金: 7.2億円」 | **強**（`_money_text` 同一） |

**再掲を削ってもユーザーが困るか**:
  - **いいえ**。MetricsRow は Scroll 上部の **SecMetricsTitle 直下**に常時表示され、
    視線の流れ上 **先に読める**。
  - club_summary は **そのさらに下**（CardWarnings / CardNext の後）にあり、
    **同じ数字を再度読ませる価値は低い**。
  - 削減後は「シーズン状態」など **MetricsRow に無い文脈**に寄せるのが妥当。

**news との間接重複**（club_summary 外だが関連）:
  - `_news_lines` 1 行目に `{rank}` と `{wins}勝{losses}敗` を再掲。
  - CardNews Theme 化済み。**club_summary 整理とは別タスク**（news 1 行化）。

6. CardTeamExtras との重複
--------------------------------------------------------------------------------
| キー | CardTeamExtras | club_summary |
|------|----------------|--------------|
| `owner_trust` | OwnerTrustRow（任意表示） | **なし** |
| `salary_cap` | SalaryCapRow（任意表示） | **行4（同一 `_salary_cap_summary` 全文）** |
| `recent_form` | RecentFormRow（任意表示） | **なし** |

**分界の現状**:
  - Extras = **ラベル付き 1 行ずつ**（オーナー信頼・キャップ・直近成績）
  - Summary = **段落リスト**（現状は Metrics 再掲 + シーズン + キャップ長文）

**salary_cap 二重表示の実態**:
  - from_python 時、**画面上に同じキャップ長文が最大 2 回**（Extras 行 + Summary 行4）。
  - mock では Extras は「キャップ余裕あり（仮）」、Summary は別ダミー文言で **不一致**。

9. home_dashboard.gd での CardSummary 表示処理
--------------------------------------------------------------------------------
- `@onready var _club_summary: Label = %ClubSummaryLabel`
- `_apply_snapshot`: `_club_summary.text = _join_lines(d, "club_summary")`
- `_join_lines(d, "club_summary")`:
  - **max_lines 引数なし** → 配列 **全行**を `・` 付きで `\n` 連結
  - （`tasks` のみ `max_lines=3`）
- **空配列**: `""` になるが **CardSummary カード自体は常時表示**（visible 制御なし）
- **エラー時** `_clear_body`: `_club_summary.text = ""`（枠は残る）

**CardSummary の visible 制御**: **なし**（CardTeamExtras / CardWarnings のみカード単位 visible）。
