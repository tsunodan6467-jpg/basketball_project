### 3.1 ロスター閲覧（`roster_view`）

- **scene構造**: `Control` + 暗 `Background` → `Margin/RootCol` → `HeaderCard` → `StatusLabel` → `Scroll/RowList`（動的行のみ）。
- **Theme**: ルート `phase4_readonly_core.tres`。**`HeaderCard` = `Phase4HeaderCard`**。ヘッダー Label は白帯向け濃色 override 済み。
- **Scroll**: **静的 Panel なし**。`roster_view.gd` が `Label.new()` で表行生成。**`Phase4OnDarkTableHead` / `Phase4OnDarkTableCell`** を `.gd` で付与（暗背景のまま）。
- **データ**: export/pytest 実績あり。DTO 安定。
- **戻り導線**: HeaderNavRow `HomeNavButton` ○。
- **見た目課題**: ヘッダーのみ Phase4 寄り。**表本体はホーム白カード群と質感が不一致**。
- **`.tscn` のみ**: ヘッダー以上の改善は **不可**（表は `.gd` 必須）。
- **リスク**: 中〜高（ゲーム中心画面・行数・列幅）。
- **1画面目適性**: 価値は最大だが **初手としては重い**。

## 6. 推奨画面の最小実装案（Cursor 依頼骨子）
================================================================================

### 目的

`facility_summary_view` の **HeaderCard + SummaryCard** を、home_dashboard と同規律で
`Phase4HeaderCard` / `Phase4SummaryCard` に限定適用し、**詳細画面 Phase4 展開の第1弾**とする。

### 変更してよいファイル

- `godot/scenes/facility_summary_view.tscn` **のみ**

### 変更してはいけないファイル

- `godot/scripts/facility_summary_view.gd`
- `godot/themes/phase4_readonly_core.tres`
- `godot/project.godot`
- `godot/data/facility_summary_mock.json` および export/tests
- README / docs（**実装コミットでは触らない**。記録は別コミット）
- 他 scene / sandbox / 既存 reports

### 実装方針
