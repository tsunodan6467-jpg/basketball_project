### 推奨タスク名

### 対象

| 種別 | パス |

| **scene** | `godot/scenes/tactics_summary_view.tscn` |
| **script** | `godot/scripts/tactics_summary_view.gd`（**第1段では変更しない**） |

### なぜ今それを選ぶのか

1. **横展開の自然な次手** — 6画面第1段完了後、残りは **3画面のみ**。**戦術**は **CardNavMenu #9（チーム列）** で未Themeのまま目立つ。
2. **確立済みテンプレの横展開** — 財務（5枚）・OM（4枚）と同じ **Header + Scroll 内複数静的 Panel** 型。**`4b43da5` / `e6acce0` の手順をほぼコピー**できる。
3. **`.tscn` のみで第1段を閉じやすい** — ルート Theme + `Phase4HeaderCard` + 6×`Phase4SummaryCard` + ラベル濃色化。**`%PlayerRolesBody` は触らない**（財務 History / OM Missions と同型）。
4. **部分 Theme 画面を避ける** — 契約人事は Header/一部カード済みで **第1段の切り分けが曖昧**。
5. **ロスター表を避ける** — **`%RowList` は `.gd` 必須**で第1段テンプレに不向き。
6. **データ安定** — `tactics_summary_readonly` + **15 pytest**。

### なぜ他案を後回しにするか

| 案 | 後回し理由 |
|----|------------|
| **A 第2段** | 6画面とも `.gd` 必須。戦術第1段後に個別判断 |
| **C 契約人事** | 部分 Theme 済み。戦術完了後 **Risk/Players の白カード化 + 必要なら `.gd` 色補正** |
| **D ロスター** | Header 済み・表は OnDark `.gd` 専用ライン |

### 変更対象ファイルの想定（実装時）

- **変更可**: `godot/scenes/tactics_summary_view.tscn` **のみ**（第1段）
- **変更不可**: `tactics_summary_view.gd`、Theme `.tres`、`project.godot`、他 scene、export / tests / mock、README / docs、reports

### 実装する場合の安全条件

1. ルート `TacticsSummaryView` に `phase4_readonly_core.tres`。
2. `HeaderCard` → `Phase4HeaderCard`（panel override 除去・SubResource 残置）。
3. **Scroll/ScrollContent 内** Overview / Attack / Defense / Rotation / PlayerRoles / Notes の **6 Panel** → `Phase4SummaryCard`（各 Title + Body Label 濃色化）。
4. **触らない**: `%PlayerRolesBody` 子構造・connection・handler・JSON キー。
5. 目視で PlayerRoles 行が薄い場合 → **別コミット**で `tactics_summary_view.gd` の `font_color` を `Color(0.16, 0.2, 0.3, 1)` へ（`6c3dc43` / `2f808e5` パターン）。

### Godot 目視確認ポイント

- CardNavMenu #9 → 戦術サマリー
- Header + 6枚白系カード / DataSourceLabel
- HomeNavButton 戻り
- PlayerRoles 動的行（薄くても第1段では可）
