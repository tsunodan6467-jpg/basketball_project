3. home_dashboard.tscn の構造
--------------------------------------------------------------------------------
【ルート】`HomeDashboard`（Control）: script のみ。Theme なし。

【背景】`Background` ColorRect（暗色 0.06,0.07,0.09）。

【ヘッダーブロック】`Margin/RootCol/HeaderCard`（PanelContainer）
  - `theme_override_styles/panel = SubResource("StyleBoxFlat_header")` … 暗めヘッダー帯。
  - `HeaderInner` VBox（separation 8）
    - `HeaderTopRow` HBox: `HeaderBadge`（chip）+ `PlaceholderStrip`（chip）… いずれも StyleBoxFlat_chip。
      子 Label: BadgeLabel / PlaceholderLabel（明系 font_color）。
    - `HeaderNavRow` HBox: **5 ボタン**（ロスター・クラブ史・順位表・日程・施設）。
      各 Button はシーン個別の theme_override なし（デフォルト Engine ボタン見た目）。
    - `ClubNameLabel`（24px 明色）, `SeasonLabel`（15px）, `DataSourceLabel`（12px 灰青）。

【ステータス】`StatusLabel`（RootCol 直下・エラー時表示）。

【スクロール本体】`Scroll` → `Inner` VBox（separation 12）
  - `CardNavMenu` PanelContainer … **StyleBoxFlat_card**（Metrics 等と同一 SubResource）。
    - `VBoxNav`: NavTitle, NavHelp, `NavColumns` HBox（separation 14）
      - `ColTeam` / `ColLeague` / `ColClub` / `ColMgmt` … 各カテゴリ Label + 2〜3 Button。
      - **経営列 ColMgmt**: 財務サマリー・オーナーミッション・契約・人事サマリー（3 ボタン）。
  - `SecMetricsTitle` Label（「主要指標」）
  - `MetricsRow` HBox: `CardDivision` / `CardRank` / `CardMoney` … いずれも **StyleBoxFlat_card**。
  - `CardTeamExtras` … StyleBoxFlat_card。内部 Row の visibility は .gd。
  - `CardWarnings` … **StyleBoxFlat_warn**（他と別 SubResource）。
  - `CardNext` / `CardSummary` / `CardTasks` / `CardNews` … いずれも StyleBoxFlat_card。

【フッター】`FooterNote` Label（RootCol 最下・手動 JSON / 非自動起動の説明）。

【StyleBoxFlat の共有】
  - **StyleBoxFlat_card**: CardNavMenu + Metrics 3枚 + Extras + Next + Summary + Tasks + News（広く共有）。
  - **StyleBoxFlat_header**: HeaderCard のみ。
  - **StyleBoxFlat_warn**: CardWarnings のみ。
  - **StyleBoxFlat_chip**: Header 内 2 チップのみ。

【接続】HeaderNavRow の 5 ボタン + CardNavMenu 内 10 ボタン → `home_dashboard.gd` の各 `_on_*`。

--------------------------------------------------------------------------------

5. 候補A：ホーム Header だけ Theme 適用
--------------------------------------------------------------------------------
内容: `HeaderCard` を `Phase4HeaderCard` に寄せ、ロスター／契約人事ヘッダー同様に
  クラブ名・シーズン・DataSource を **白パネル上の濃色**へ揃える。チップは `Phase4ChipPanel` 可。

利点:
  - **「顔」となる帯だけ**先に本番 Theme と整合。商品価値 / Steam 第一印象に直結。
  - Metrics / Nav / Footer は現状の暗色カードのまま → **二層 UI（上白・下暗）**はロスター画面と同型で説明しやすい。
  - HeaderNavRow は **兄弟のまま**残せる。ボタン数・接続・シグナルは変更不要（見た目のみ変化の可能性）。

欠点:
  - ヘッダー内 Label の **per-node theme_override_colors を大量に差し替え**（契約人事と同程度の作業）。
  - `StyleBoxFlat_header` を外すと、チップ以外の「暗ヘッダー一体感」は失われる（意図的な移行）。

触りそうなファイル:
  **主に** `godot/scenes/home_dashboard.tscn`。**任意**で `phase4_readonly_core.tres`（新 variation は今回禁止方針なら触らない）。

**推奨パターン（調査メモ）**: ルートではなく **`HeaderCard` ノードにのみ `theme = phase4_readonly_core` を割り当て**、
  当該 Panel に `theme_type_variation = Phase4HeaderCard` を指定すると、**Scroll 以下には Theme が伝播しない**
  （子孫のみ適用）ため、Metrics / CardNavMenu の SubResource スタイルと **衝突しにくい**。

.tscn だけで済むか: **はい**（現行 .gd との独立性が高い）。

リスク:
  - HeaderNavRow の Button が **Theme 既定の明色ボタン**になり、従来のフラット暗色と質感が変わる（好み・コントラスト要確認）。
  - UID はノード追加なし・プロパティ変更中心なら中程度。

観点 1〜14: 高 / 高 / 中（帯はモダン寄り。90年代風は下層カードの方が効きやすい） / 低 / 中（見た目のみ） / 無 / 1 ファイルに集約可 / Yes / 不要 / 無（.gd 不変） / 中 / 中 / 高 / **最有力の第一歩**

7. 候補C：経営カテゴリカードだけ Theme 適用

11. StyleBoxFlat 衝突・削除リスク
--------------------------------------------------------------------------------
- **HeaderCard**: `StyleBoxFlat_header` をやめて `Phase4HeaderCard` に置換する場合、
  **ヘッダー内の全 Label 色を白パネル前提に再設定**しないと可読性が崩れる（ロスター／契約人事と同パターン）。
- **StyleBoxFlat_card**: 1 枚だけ Phase4SummaryCard に差し替え**可能**だが、**他ノードは同じ SubResource のまま**
  残るため、「SubResource を削除」すると **複数カードが同時に壊れる**。削除は **共用が解けるまで不可**。
- **StyleBoxFlat_warn**: Phase4WarningCard と意味が近いが、現行 warn は **赤茶系ダーク**。Theme の Warning は **黄系ライト**。
  置換時は **文言色（HWarn / WarningsLabel）**の再調整が必要。
- **StyleBoxFlat_chip**: Phase4ChipPanel で置換可能。ヘッダー帯とセットで整えるとよい。

結論: **SubResource は「参照ゼロになったら」削除**。部分適用では **残す**のが安全。


12. HeaderNavRow / NavColumns への影響
--------------------------------------------------------------------------------
- HeaderNavRow は **HeaderCard 直下**。HeaderCard に局所 Theme を付けると **ボタンもその Theme 配下**になる。
- NavColumns は **Scroll 内 CardNavMenu** にあり、HeaderCard 局所 Theme の外に置ける → **A で Nav は無変更可能**。
- ルートへ `theme` を付けると **全画面要素に Button/Label 既定が波及** → 非推奨（F に近い）。


13. from_python / mock / DataSourceLabel への影響
--------------------------------------------------------------------------------
- 表示文字列はすべて `_data_source_caption` と各 `_txt` / `_join_lines`。**Theme は文字列を変えない**。
- DataSourceLabel はシーン上の Label。**色・フォントを Theme variation に寄せるだけ**なら .gd 不要。
- pytest（home_dashboard export）は **JSON 構造・パス**を見るため、**シーン見た目のみの変更で赤化する可能性は低い**（実装時も確認推奨）。

17. 今すぐ実装してよい最小案（方針のみ）
--------------------------------------------------------------------------------
- **ステップ A0**: `HomeDashboard` ルートには Theme を付けない。
- **ステップ A1**: `HeaderCard` に `theme = phase4_readonly_core.tres` を割当（ext_resource 追加）。
- **ステップ A2**: `HeaderCard` の Panel に `theme_type_variation = Phase4HeaderCard`。
  `theme_override_styles/panel`（StyleBoxFlat_header）を外す。
- **ステップ A3**: ClubNameLabel / SeasonLabel / DataSourceLabel / Badge / Placeholder を
  ロスター・契約人事ヘッダーと同様 **濃色・Phase4DataSource 等**に整理（必要なら `theme_type_variation`）。
- **ステップ A4**: Godot 4.6.2 で HeaderNavRow 5 ボタンの視認性・フォーカス・ホバーを確認。

- **ステップ B 最小**: `CardDivision` のみ `theme_type_variation = Phase4SummaryCard`、
  `HDiv` → Phase4Muted 相当、`DivisionLabel` → Phase4Title 相当の濃色（白上可読）。
