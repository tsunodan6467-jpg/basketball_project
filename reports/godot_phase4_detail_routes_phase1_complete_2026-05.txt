3. 「Godot詳細導線第1段完了」の定義
================================================================================

**Godot詳細導線第1段完了** とは、以下を満たす状態と定義する。

1. 一覧画面から専用詳細画面へ遷移できる
2. 詳細画面は **Autoload payload のみ** を読む（scene 内 JSON 再読込なし）
3. **JSON / save / Python 自動起動は行わない**
4. **読み取り専用** を維持する
5. 既存の一覧内 **「詳細」** 表示は消さない（詳細=インライン / 画面=専用 scene の併存）
6. 専用詳細画面から **一覧へ戻れる**（return_scene + BackButton）
7. 実害が大きい **日程・ロスター** ではスクロール位置も復元される
8. **player_detail / game_detail** の最小 UI 土台ができた

**本プロジェクトは上記条件を満たしたため、Phase D は「詳細導線第1段完了」として扱う。**

6. 現在の安全設計
================================================================================

（readonly_selection_context.gd / 各 view .gd の grep 確認に基づく）

### ReadonlySelectionContext

- **選択状態**（kind / payload / return_scene / source_label / player_id / event_id）
  と **return_state**（scene_path → Dictionary）を **分離**
- return_state は scene_path ごとに保持（`_return_state_by_scene`）
- `consume_return_state(scene_path)` で **一度きり復元**
- `clear()` 引数なし: 従来どおり **全消去**（選択 + return_state）
- `clear(true)`: 選択状態のみ消し **return_state は保持**
- `has_selection()` は return_state に **影響しない**

### 各画面の共通パターン

- Autoload 取得: `get_node_or_null("/root/ReadonlySelectionContext")` + `ctx.call(...)`
- **ReadonlySelectionContext. 直接参照禁止**（player_detail / game_detail / 接続元すべて）
- Godot から **Python 自動起動なし**
- **save 変更なし**
- **JSON 構造変更なし**（Phase D 期間）
- **export 変更なし**（Phase D 期間）
- 各実装タスク後 **pytest 79 passed** を維持（Python 回帰）

### 詳細画面 Back / Home

| ボタン | 呼び出し | 意味 |
|--------|----------|------|
| BackButton | clear(true) | 一覧 scroll 復元のため return_state 保持 |
| HomeNavButton | clear() | stale state を残さない全消去 |
