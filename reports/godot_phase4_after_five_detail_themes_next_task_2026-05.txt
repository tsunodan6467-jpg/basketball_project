| 種別 | パス |

| **script** | `godot/scripts/owner_mission_view.gd`（**第1段では変更しない**） |

### なぜ今それを選ぶのか

1. **横展開の自然な次手** — 5画面完了後、残り4画面のうち **未Theme で経営系の本命**（`35eeed5` / `99c279d` で財務の次は OM と整理済み）。
2. **財務サマリーとの構造一致** — **Header + Scroll/ScrollContent 内複数静的 Panel**（財務5枚 → OM **4枚**）。**`4b43da5` の手順をほぼコピー**できる。
3. **`.tscn` のみで第1段を閉じやすい** — Trust / Eval / Caution は静的 Label。**`MissionsCard` パネルだけ Phase4SummaryCard 化**し、**`%MissionsBody` 子の動的生成は触らない**（財務の HistoryCard と同型）。
4. **既知リスクの先読み** — `MissionsBody` の `Label.new()` は **財務履歴と同じ明色**のため、第1段目視後に **`6c3dc43` 型の色補正**が別コミットで必要になる可能性が高い（**第1段の範囲外で許容**）。
5. **データ・導線** — **CardNavMenu #8**。**`owner_mission_readonly` + 13 pytest** 安定。

### なぜ他案を後回しにするか

| 案 | 後回し理由 |
|----|------------|
| **A 第2段** | 5画面とも `.gd` 必須。OM 第1段で横展開を続ける |
| **C 戦術** | 静的6枚で差分大。OM で経営系テンプレ確定後 |
| **D 契約人事** | 部分 Theme 済み・Risk/Players は `.gd` |
| **E ロスター** | Header 済み・表は別フェーズ |

### 変更対象ファイルの想定（実装時）

- **変更可**: `godot/scenes/owner_mission_view.tscn` **のみ**（第1段）
