## 5. 推奨する次タスク
================================================================================

### 推奨タスク名

**財務サマリー閲覧 — Phase4 Theme 第1段（HeaderCard + Scroll 内静的カード5枚）**

### 対象

| 種別 | パス |
|------|------|
| **scene** | `godot/scenes/finance_summary_view.tscn` |
| **script** | `godot/scripts/finance_summary_view.gd`（**変更しない**） |

### なぜ今それを選ぶのか

1. **横展開継続の自然な次手** — 4画面第1段完了後、残り **5画面**のうち **未Theme が最も目立つ経営系本命**（`23a8fcf` 第2候補、`a5eec31` で日程の次は財務と整理済み）。
2. **`.tscn` のみで第1段を閉じやすい** — ルート Theme + `HeaderCard` → `Phase4HeaderCard` + **FinanceCard / PriorCard / SalaryCard / HistoryCard / CautionCard** → `Phase4SummaryCard`（**日程の Scroll内カード型と同系**）。**`%Card*Body` / タイトル Label** は `.gd` 参照済みのまま text 更新のみ。
3. **第2段の切り分けが明確** — **`%HistoryBody` 内の `Label.new()` 履歴行**のみ暗地のまま第2段。**他4カード + Header は静的**で今回完了可能。
4. **ユーザー価値・データ安定** — **CardNavMenu #7** 主入口。**`finance_summary_readonly` + 10 pytest**。
5. **見た目インパクト** — HeaderNavRow 系4画面は完了済み。**経営列の代表画面**が暗色のまま残るため、**ホームから入ったときの改善が大きい**。
6. **テンプレ拡張** — 成功後、**オーナーミッション**（4枚・Missions 動的は第2段）へ **同型コピー**しやすい。

### なぜ他案を後回しにするか

| 案 | 後回し理由 |
|----|------------|
| **A 第2段** | 4画面とも `.gd` 必須。横展開テンプレを **5画面目（財務）** で広げてから |
| **C OM** | 財務で **Scroll内複数静的Panel** の手順を確定してから |
| **D 戦術** | 静的カード **6枚**で1コミットが重い。将来 UI 変更余地 |
| **E 契約人事** | 部分 Theme 済み。**Risk/Players は `.gd` 必須**の続き |
| **F ロスター** | ヘッダー済み。**表は別フェーズ** |

### 変更対象ファイルの想定（実装時）

- **変更可**: `godot/scenes/finance_summary_view.tscn` **のみ**
- **変更不可**: `finance_summary_view.gd`、`phase4_readonly_core.tres`、`project.godot`、他 scene、export / tests / mock、README / docs、reports

### 実装する場合の安全条件

1. ルート `FinanceSummaryView` に `phase4_readonly_core.tres`。
2. `HeaderCard` → `Phase4HeaderCard`（panel override 除去・SubResource 残置）。
3. **Scroll/ScrollContent 内**の **FinanceCard / PriorCard / SalaryCard / HistoryCard / CautionCard** のみ → `Phase4SummaryCard`（**各 VBox 内 Title + Body Label 濃色化**）。
4. **`NotesFooterLabel`** を白カード向け muted 濃色（必要なら）。
5. **触らない**: `HistoryBody` 子ノード構造（`.gd` が `Label.new()` する先）、connection、handler、JSON キー。
6. Godot 保存後 **tracked 差分は当該 `.tscn` のみ**。
