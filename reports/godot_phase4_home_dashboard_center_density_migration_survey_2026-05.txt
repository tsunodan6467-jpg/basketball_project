  「1 行に見える」保証は弱いが、**clip や custom_minimum_size** で物理的に抑えるのは

16. CardNews の 1 行化に必要な変更
--------------------------------------------------------------------------------
- **現状**: `_news.text = _join_lines(d, "news")` で **max_lines=0** → 配列全件。
- **export 側**: `_news_lines` は既に `max_news`（既定 3）で **最大 3 要素**。
  それでも Godot 上は最大 3 行の「・」箇条書きになりうる。**「1 行ヘッドライン」には未達**。
- **最小変更（推奨の第 1 手）**: `home_dashboard.gd` で
  `_join_lines(d, "news", 1)` に変更、または先頭要素のみを取り出して 1 行表示。
  → **JSON キーは不変**、pytest（キー存在検査中心）も通しやすい。
- **export のみ max_news=1**: JSON 上の news が短くなる。**他ツールが同 JSON を
  フル利用する場合は不利**。Godot 専用なら可。
- **正攻法**: `news_headline`（string）を追加し、既存 `news`（array）は将来の詳細用に残す。
  → **キー追加・pytest・mock・gd 全て更新**（案 F の一部）。

**結論（後述「Cursor 推奨」参照）**: まず **`.gd` で表示を 1 行に制限**するのがコスト最小。
専用キーは中長期。

18. CardSummary / club_summary の扱い
--------------------------------------------------------------------------------
- export の `_club_summary_lines` は **順位・戦績・ディビジョン/資金・シーズン状態・
  キャップ要約**等の **説明用複数行**。sandbox CardClubState の「サラリー余力／士気」
  1 行要約とは **役割が一致しない**。
- **表示だけ 1 行に切る**（gd で先頭行のみ）は技術的には可能だが、**残る 1 行目は
  多くの場合「順位・戦績: …」**で sandbox のクラブ状態とは異なる。
- **sandbox 相当の 1 行要約**は **現行 DTO のみでは不十分**。`morale_short` 等の
  新規 or export の再構成が必要（案 C・F）。

19. CardRank / rank_record の DTO 課題
--------------------------------------------------------------------------------
- 現行: `rank_record = f"{rank} / {wins}勝{losses}敗"` または順位未確定版。
- sandbox の「PO圏まで 2.0差 / 直近5試合 3勝2敗」は **別計算・別文脈**。

- **単一キー rank_record だけを書き換えて擬似的に出す**ことは、**テスト期待や
  他画面との整合を壊しやすい**。
- **結論**: **現 DTO のみで立ち位置判断文は再現できない**（export 拡張 or 新キー必須）。

20. CardNavMenu / HeaderNavRow / 将来左レールの関係
--------------------------------------------------------------------------------
- **HeaderNavRow（5）**: ロスター・クラブ史・順位表・日程・施設への最短ボタン。
- **CardNavMenu（9）**: 上記に加え **戦術・財務・オーナーミッション・契約人事** 等、
  Header に無い導線を提供。**完全重複ではない**（Header は頻出サブセット）。
- **左レール未実装の現状**: CardNavMenu は **「全機能への地図」**として依然有用。
  **左レールを「遷移あり」で入れる場合**のみ、CardNavMenu の縮小・1 行化・削除が
  論点になる。**見た目のみの左レール**なら三重化は緩和されるが説明コストは残る。
- **三重ナビ回避策**:
  - 左レールは **カテゴリのみ**（sandbox 現状）＋実遷移は Header/CardNav に一本化、または
  - 左レールに実遷移を載せるなら **HeaderNavRow の縮退・CardNavMenu の統合**を
