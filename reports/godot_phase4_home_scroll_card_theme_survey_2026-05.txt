3. home_dashboard.tscn の Scroll 以下構造
--------------------------------------------------------------------------------
【階層】`Margin/RootCol/Scroll` → `Inner`（VBox・separation 12）

【1】`CardNavMenu`（PanelContainer）
  - `theme_override_styles/panel = SubResource("StyleBoxFlat_card")`
  - 子: `VBoxNav` → `NavTitle`, `NavHelp`, **`NavColumns`**（HBox）
    - `ColTeam` / `ColLeague` / `ColClub` / `ColMgmt` … 各カテゴリ Label + 2〜3 **Button**（MenuCard*）
    - **経営 3 ボタン**は `ColMgmt` 内

【2】`SecMetricsTitle`（Label・「主要指標」）— カード外タイトル

【3】`MetricsRow`（HBox）
  - **`CardDivision`** / **`CardRank`** / **`CardMoney`** … いずれも Panel + **同一** `StyleBoxFlat_card`
  - 各カード内: 見出し Label（HDiv 等）+ 値 Label（`DivisionLabel` 等・`unique_name_in_owner`）

【4】`CardTeamExtras`（Panel + `StyleBoxFlat_card`）— visibility は .gd

【5】`CardWarnings`（Panel + **`StyleBoxFlat_warn`**）— 警告系配色。他カードと SubResource が異なる

【6】`CardNext` / `CardSummary` / `CardTasks` / `CardNews` … いずれも **`StyleBoxFlat_card`**

【共有の実態】
  - **`StyleBoxFlat_card`**: 上記 **9 箇所**の PanelContainer が **同一 SubResource ID** を参照。
  - **1 枚だけ** `theme_override_styles/panel` を外し、`theme_type_variation = Phase4SummaryCard` 等に差し替えても、**他 8 箇所が参照を残す限り SubResource を削除してはならない**。
  - **`StyleBoxFlat_warn`**: `CardWarnings` のみ。


4. home_dashboard.gd の Scroll 以下更新処理
--------------------------------------------------------------------------------
- `_apply_snapshot`: `%DivisionLabel` / `%RankRecordLabel` / `%MoneyLabel` / `%NextGameLabel` /
  `%ClubSummaryLabel` / `%TasksLabel` / `%NewsLabel` などに **文字列のみ**代入。
- `_join_lines` で tasks / news / club_summary は **複数行テキスト**（先頭に「・」）。
- `_set_optional_row` で `CardTeamExtras` 内の Row visible と値 Label。
- **動的ノード生成なし**。**`add_theme_color_override` 等のコード上書きなし**。
- **結論**: Scroll 以下の Theme 試験は **原則 `.tscn` のみで完結可能**（白カード化時は **子 Label の濃色化**をシーンで足す）。


5. 候補A：Metrics 系カード 1 枚（例 CardDivision）
--------------------------------------------------------------------------------
利点:
  - **主要指標**はデモで目立つ。商品価値が高い。
  - 同一 `MetricsRow` 内の **1 枚だけ**変えると、Before/After が並んで比較しやすい。

欠点:
  - 隣の `CardRank` / `CardMoney` が暗のままだと **質感の段差**が出る（意図的なら可）。
  - 値ラベルは現状 **明色大文字**（17px 付近）→ 白パネルでは **必ず濃色へ**（調整行数は少ないが必須）。

触りそうなファイル: `godot/scenes/home_dashboard.tscn` のみ（+ 既存 ext_resource Theme は流用）。

.tscn のみ: **はい**（推奨パターン: **対象 Panel のみ** `theme = ExtResource("2_theme")` + `theme_type_variation = Phase4SummaryCard`、
  `theme_override_styles/panel` を当該ノードから削除。子 Label に Phase4Title / Phase4Muted 等または `theme_override_colors`）。

StyleBoxFlat 共有リスク: **低**（当該 Panel だけ参照を外す。**SubResource は削除しない**）。

文字色リスク: **中**（見出し 2 + 値 1 程度の最小セット）。

観点 1〜12: 高 / 高 / Yes / 不要 / 無 / 無 / 無 / 低 / 小 / 高 / 高 / **第一候補**

12. StyleBoxFlat 衝突・削除リスク
--------------------------------------------------------------------------------
- **削除してはならないタイミング**: いずれかのノードがまだ `SubResource("StyleBoxFlat_card")` を参照している間。
- **1 カード試験の安全手順**: 対象 Panel だけ **参照を外す** → 必要なら **その Panel にだけ** `theme = ExtResource("2_theme")` を重ねる（ファイル先頭の ext_resource は **既に存在**）。
- **CardWarnings**: `Phase4WarningCard` と意味は近いが **色相が異なる**（warn SubResource は赤茶ダーク）。置換時は **HWarn / WarningsLabel** の再調整が必要 → **2 枚目以降の候補**向き。


13. .gd 変更が必要になりそうな箇所
--------------------------------------------------------------------------------
- **現状の Scroll 以下 Theme 試験では不要**。
- 将来、**JSON 由来の文字列長に応じて font_size を変える**等を入れる場合のみ .gd が絡む。


14. HeaderNavRow / NavColumns / 導線への影響
--------------------------------------------------------------------------------
- **HeaderNavRow** は Scroll の外（HeaderCard 内）。Scroll 以下の編集では **接続・ノード数に触れなければ無影響**。
- **NavColumns** は `CardNavMenu` 内。D/E を避ければ **ボタン接続はそのまま**。
- **`change_scene_to_file` の connection** は Scroll 内 Button から HomeDashboard 根。パス文字列が変わらなければ **維持**。

18. 今すぐ実装してよい最小案（方針のみ）
--------------------------------------------------------------------------------
- **`CardDivision` のみ**: `theme = ExtResource("2_theme")`、`theme_type_variation = &"Phase4SummaryCard"`、
  当ノードの `theme_override_styles/panel` を削除。`HDiv` を `Phase4Muted` 系または濃灰 override、`DivisionLabel` を `Phase4Title` 系相当の濃色に。
- **SubResource `StyleBoxFlat_card` は残す**（他 8 参照のため）。
- **コミット 1 本・差分レビューで丸ごと revert 可能**。


19. まだ実装しない方がよいこと
--------------------------------------------------------------------------------
- `StyleBoxFlat_card` を **参照ゼロになるまで消す**こと。
- `Inner` や `Scroll` に **ルート相当の Theme** を付けて **一括波及**させること。
- **CardNavMenu 全体**を最初の 1 歩にすること。
