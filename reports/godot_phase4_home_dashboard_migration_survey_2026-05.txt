3. 本線 home_dashboard.tscn の現状構造
--------------------------------------
ルート: **HomeDashboard**（Control）→ **Background** → **Margin** → **RootCol**（VBox）

- **HeaderCard**（PanelContainer・Theme 限定適用済みの想定）
  - **HeaderInner**（VBox）
    - **HeaderTopRow**: **HeaderBadge**（仮バッジ） / **PlaceholderStrip**（仮ストリップ）
    - **HeaderNavRow**（HBox）: **5 ボタン**（ロスター・クラブ史・順位表・日程・施設）
    - **ClubNameLabel**, **SeasonLabel**
    - **DataSourceLabel**（読込元表示）
- **StatusLabel**（読込失敗時など・RootCol 直下）
- **Scroll** → **Inner**（VBox）
  - **CardNavMenu**: カテゴリ別ボタン列（チーム / リーグ / クラブ / 経営）
  - **SecMetricsTitle** + **MetricsRow**: **CardDivision**（division） / **CardRank**（rank_record） / **CardMoney**（money）
  - **CardTeamExtras**: owner_trust / salary_cap / recent_form（任意行表示）
  - **CardWarnings**: warnings
  - **CardNext**: next_game
  - **CardSummary**: club_summary（複数行を1ラベルに結合表示）
  - **CardTasks**: tasks（最大 3 行・`_join_lines(..., 3)`）
  - **CardNews**: news（**行数上限なし**で `_join_lines` — 配列全要素を「・」で連結）
- **FooterNote**（補足文言）

sandbox との対応イメージ: 本線は **Header 内に Nav + クラブ名 + シーズン**、指標は **Scroll 内 Metrics 3 カード**に分割。sandbox は **ClubBand に指標を集約**した帯＋**中央 2 カラム**。


4. 本線 home_dashboard.gd の現状機能
------------------------------------
- **`_home_json_candidate_paths`**: `home_dashboard_from_python.json` → `home_dashboard_mock.json` の順で探索。
- **`_load_home_snapshot`**: ファイル存在・parse・型チェック。失敗時 `{"_error": ...}`。
- **`_apply_snapshot`**: 失敗時は Status 表示・本文クリア。成功時はキーごとに Label へ代入:
  - スカラー: `club_name`, `season_label`, `division`, `rank_record`, `money`, `next_game`
  - **`_join_lines`**: `club_summary`（全行）、`tasks`（**max 3**）、`news`（**max 0 = 制限なし**で全行）
  - 任意行: `owner_trust`, `salary_cap`, `recent_form`, `warnings` → 空なら行非表示。CardTeamExtras / CardWarnings の visible 連動。
- **`change_scene_to_file`**: 10 画面それぞれ定数パスへ。コメントで閲覧専用であることを明示。
- **`_data_source_caption`**: from_python と mock で文言切替。

**守るべき契約**: `REQUIRED_KEYS`（pytest）と **Godot mock のキー集合**が export の **`build_home_dashboard_readonly_dict` の戻り値**と一致していること。

- **RightStack**: **ClubBand**（ロゴ枠 SG/LOGO + クラブ名 + シーズン + 地区/戦績 + 資金/信頼）→ **MidScroll** → **BottomStrip**

7. 本線へ移す候補要素（優先度の高い順・論理単位）
-----------------------------------------------
1. **視覚・情報密度（Theme / variation / フォント・余白）** — キー不変で `.tscn` のみまたは軽微なラベルプロパティ。
2. **CardTasks 主役化** — `Phase4WarningCard` 等の視覚階層は `.tscn` で可能。文言内容は **tasks 配列**に依存。
3. **Metrics 3 カード（Division / Rank / Money）** と sandbox **ClubBand** の**情報役割の整理**（重複削減は設計・DTO 両面）。
4. **PlaceholderStrip → ロゴ枠風パネル**（sandbox の SG/LOGO 相当）— **SubResource** 追加は可能だが UID/差分運用に注意。
5. **BottomStrip 相当** — 本線は **FooterNote** あり。役割を統合するか検討。
6. **左レール** — **導線・ドキュメント・HeaderNavRow との関係**が大きいため後段。


8. 本線へまだ移さない方がよい要素
-------------------------------
- **左レールをそのまま本線に足すこと**（案 C）— HeaderNavRow + CardNavMenu と**三系統ナビ**になりやすい。
- **HeaderNavRow をいきなり削除**すること — 10 画面への**最短導線**が失われ、各サブ画面の「ホームへ戻る」との**認知モデル**も変わる。
- **ClubBand を一塊で本線に載せ替え**て Metrics を廃する等の**大レイアウト変更**（案 D）— 回帰範囲が広い。
- **JSON キー追加を伴わないまま**、sandbox 固定文言だけを本線に焼き付けること — **from_python と乖離**し、Steam 品質の観点でも不採用。


9. 案 A: Header だけ本番寄せ
----------------------------
- **安全性**: 高（範囲が HeaderCard 内に閉じやすい）。
- **実装範囲**: `.tscn` 中心。ロゴ枠・行レイアウト調整。
- **10 画面導線**: **HeaderNavRow を触らなければ影響小**。
- **from_python/mock**: キー不変なら **影響なし**。
- **home_dashboard.gd**: **原則不要**（DataSourceLabel 位置は `.tscn` で調整可）。
- **DTO/Python**: 不要（見た目のみの場合）。
- **Theme**: 既存 variation の利用で **Theme ファイル変更なし**も可能。
- **HeaderNavRow**: **維持**前提が最も整合。
- **左ナビ**: 未導入のまま。
- **1280×720**: Header が縦に伸びすぎないよう注意。
- **本命一致度**: 中（ClubBand 全体再現ではない）。
- **戻しやすさ**: 高。
- **pytest**: キー・export 不変なら **影響なし**。
- **今すぐ実装**: **最有力の第一歩**。


10. 案 B: 中央カード密度だけ移植
--------------------------------
- **安全性**: 中（Scroll 内のみだが、**表示内容は JSON 駆動**）。
- **文言の sandbox 寄せ**:
  - **rank_record を「PO圏まで…」型にする**: 現行 export は **`{rank} / {wins}勝{losses}敗`**。**プレーオフ差分**は **別ロジック or 新キー**が必要。**現行 DTO のみでは出せない**。
  - **news 1 行**: `_join_lines(d, "news")` は **全行連結**。**1 行だけ**にするには (1) `home_dashboard.gd` で先頭のみ表示 (2) export の `max_news=1` (3) 新キー `news_headline` — (2)(3) は **Python + pytest 更新**。
  - **club_summary 1 行要約**: 現行は **複数行の配列**。**短い1行**は export の `_club_summary_lines` 変更 or 新キー or gd で結合短縮。**現状の mock 構造のままでは sandbox 固定文と一致しない**。
- **tasks 主役化**: 並び順・件数は `_task_lines` が支配。**カードの視覚強調**は `.tscn` のみで可能。
- **pytest**: Python 変更時は **必ず再実行**。

- **今すぐ本線で消してよいか**: **いいえ**。
- **矛盾**: 左レールが「ホーム/チーム/…」見た目のみで、実際の遷移は HeaderNavRow + カード — **説明コストと誤タップ**が増える。

- **結論**: **現時点では非推奨**。

16. HeaderNavRow の扱い（結論寄り）
----------------------------------
- **本線に左レールを入れる場合**: 原則 **HeaderNavRow は残す**。左レールは**装飾＋将来のカテゴリ表示**に留めるか、**クリック可能にするなら**遷移先を HeaderNavRow / CardNavMenu と**重複させない**設計が必要。
- **HeaderNavRow を消す判断**: **今はしない**。ドキュメント・ユーザー教育・サブ画面ヘッダー文言の一括見直しが前提。


17. from_python/mock 維持の注意
-------------------------------
- **キー集合を変える場合**は必ず **export・mock・REQUIRED_KEYS・`_apply_snapshot`** を同じコミット系列で更新。
- **`news` の全行連結**は、行数が増えると **CardNews が縦に伸び Scroll が圧迫** — 1 行化は **gd か export のどちらか**で明示的に制限する。


18. DataSourceLabel の置き場所
------------------------------
- 現状: **HeaderCard / HeaderInner 内**、シーズンラベル付近。**読込元の可視化**として妥当。
- sandbox ClubBand 化でヘッダが縦長になる場合: **Header 最下段**に固定するか、**Scroll 外の薄い一行バー**に分離する案もある（実装時判断）。**Steam 品質**: 「モックか本物か」が一目で分かることが重要。


19. CardStandings / CardNews / CardClubState の DTO 課題（本線との対応）
--------------------------------------------------------------------
※ 本線には **CardStandings 専用ノードはない**。相当は **MetricsRow の CardRank（rank_record）** と **club_summary**。

- **sandbox の「PO圏まで 2.0差 / 直近5試合 3勝2敗」**:
  - **rank_record** 1 本で表現するには **export 側でプレーオフ差・直近成績を計算して文字列化**する必要がある。**現行 `build_home_dashboard_readonly_dict` では未提供**（`recent_form` は別キーで存在）。
- **CardNews 1 行ヘッドライン**:
  - `news` は配列。**先頭 1 要素だけを見出しに**するのが最も変更が小さい（gd 1 行 or export で 1 要素配列）。
- **「サラリー余力あり / 士気良好」**:
  - **salary_cap** に近い文言は export 可能。**士気**は現行キーに**専用フィールドがない**。**morale_short 等の新キー**か、**club_summary の先頭行に統合**か、**owner_trust 以外の派生**かを決める必要あり。
