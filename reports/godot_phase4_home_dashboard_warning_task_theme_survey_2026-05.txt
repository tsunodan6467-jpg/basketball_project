6. home_dashboard.gd での warnings / tasks 処理
--------------------------------------------------------------------------------
- **読込**: `_home_json_candidate_paths`（`from_python` 優先 → mock）。変更なし前提の調査。
- **tasks**: `_tasks.text = _join_lines(d, "tasks", 3)` — **最大 3 行**、先頭に「・」連結。
  **空配列なら空文字**。
- **warnings**: `_set_optional_row(_warnings_row, _warnings, _txt(d, "warnings"))`
  - `_txt`: キー欠如・null は **空文字**。
  - `_set_optional_row`: **text が空なら `row.visible = false`**、非空なら `visible = true`
    かつ `value_label.text = text`。
- **カード表示**:
  - `_warnings_card.visible = _warnings_row.visible` — **警告文が空のときカードごと非表示**。
  - `_clear_body` でも **`_warnings_card.visible = false`**。
- **CardTasks**: **visible 制御なし**（常に表示カードとして存在）。
- **Theme 変更との独立性**: **visible はスクリプトが `PanelContainer` / `HBoxContainer` に
  対して設定**。Theme の panel Style の差し替えと **競合しない**（Godot の通常動作）。

8. CardWarnings と CardTasks の役割分界（情報設計）
--------------------------------------------------------------------------------
- **CardWarnings**: **単一文字列 `warnings`（任意表示）**。UI ラベル「注意 / 警告」。
  **リスク・コンプライアンス・緊急度が高いメッセージ**向けの枠として自然。
- **CardTasks**: **配列 `tasks` 最大 3 件**。「今やること」＝ **行動キュー / ToDo**。
  **プレイヤーが次に着手する操作の候補**に近い。
- **Tk 由来の整理**（情報設計メモ）: ホームは「次に何をするか」が主題。タスクは **行動**、
  警告は **状態の異常・注意**。同一トーンにすると **ユーザーが「どちらを先に読むべきか」
  を判別しづらく**なる。
- **推奨する分界**:
  - **視覚**: **注意＝Warning 系（Phase4WarningCard または現行の暗 warn 帯のどちらか一系統）**、
    **ToDo＝中立〜前向き（SummaryCard または現行の暗 card のまま）**に寄せるのが読みやすい。
  - **両方を Phase4WarningCard にするのは非推奨**（意味は違っても「全部注意色」に見える）。

9. 案A — CardWarnings を次に WarningCard 化する案
--------------------------------------------------------------------------------
- **内容**: CardWarnings に `theme` + `Phase4WarningCard`、panel の `StyleBoxFlat_warn`
  override を外す。**HWarn / WarningsLabel を白〜クリーム地に合う濃色／Phase4NoteText 系に
  寄せる**（CardNews と同様に **子ラベルの override が必須**）。
- **評価（調査観点 1〜18 の要約）**:
  1. 安全性: **高**（1 カード・他カードの `StyleBoxFlat_warn` 参照なし）。
  2. .tscn のみ: **可能**（ラベル色も .tscn で完結）。
  3. .gd 変更: **不要**。
  4. Python/export/DTO: **不要**。
  5. Theme .tres 変更: **不要**（既存 variation のみ）。
  6. StyleBoxFlat_warn: **CardWarnings から外すだけで SubResource は残せる**。
  7. visible: **影響なし**（`_warnings_card.visible` は従来どおり）。
  8. from_python/mock: **不要**。
  9. HeaderNavRow / 10 画面: **影響なし**。
  10. 1280×720: **要 Godot 目視**（暗 warn → ライト WarningCard でコントラストが変化）。
  11. CardNews/CardNext: **白系カードが縦に増える**が、**「注意」ブロックが明確になる**利点。
  12. sandbox: **「注意」カードを強調する方向とは一致しやすい**（sandbox に本線同型の
      CardWarnings が無い点は差分）。
  13. 意味の分かりやすさ: **向上しやすい**（契約人事の「注意」と同 variation で統一）。
  14. 戻しやすさ: **高**（1 コミット・panel override 戻しで復旧可）。
  15. pytest: **影響なし**（export は JSON のみ検証する前提）。
  16. 手動確認: **必須**（warnings 非空 mock での表示・空での非表示）。
  17. Steam 品質: **一貫した読み取り UI**としてプラス。
  18. 今すぐ実装: **候補として妥当**（下記「結論」参照）。

10. 案B — CardTasks を次に WarningCard 化する案

- **結論**: **単独での「次の 1 コミット」としては非推奨**。
